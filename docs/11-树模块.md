# 04 - 树模块需求文档

## 1. 模块概述

树是计算机科学中最核心的非线性数据结构之一，广泛应用于数据库索引（B+树）、文件系统（目录树）、编译器（语法树）、网络路由（Trie）、内存管理（红黑树）等领域。本模块旨在系统性地覆盖从基础二叉树到高级树结构的全部知识体系，提供完整的 C/C++ 实现与 Python 辅助验证。

当前项目树模块代码集中在 `Tree树/` 和 `基础部分框架－3/` 目录下，存在代码重复、功能缺失、质量参差不齐等问题，亟需系统性重构与扩展。

> **Python 实现**：参见 `python/algorithms/data_structures/` 中的 binary_tree.py、binary_search_tree.py、avl_tree.py、trie.py

### 2.2 逐文件详细分析

#### 2.2.1 `Tree树/test.cpp` — 全功能二叉树

**功能概述**：基于 `BTNode` 结构体（`char data`）实现的二叉树，包含创建、查找、销毁、求高度、输出、递归遍历（先/中/后序）、层次遍历，菜单提供 A-N 共 14 个选项。

**具体 Bug 与问题**：

1. **非递归遍历未实现（严重）**：菜单选项 G（先序非递归）、I（中序非递归）、K（后序非递归）均直接调用对应的递归函数，属于功能造假。例如：
   - 选项 G 调用 `preOrder(BT)` 而非非递归版本
   - 选项 I 调用 `inOrder(BT)` 而非非递归版本
   - 选项 K 调用 `postOrder(BT)` 而非非递归版本

2. **哈夫曼树未实现（严重）**：菜单选项 M（递归构造哈夫曼树）、N（非递归构造哈夫曼树）的 case 分支为空，仅有注释。

3. **队列实现缺陷**：
   - `SqQueue` 使用固定数组 `BTNode* data[MAX_SIZE]`，`front` 和 `rear` 初始化为 `-1`，采用非循环队列实现
   - `enQueue` 仅判断 `rear == MAX_SIZE - 1`，未考虑 `front` 前方的空间浪费（假溢出）
   - 注释错误：`enQueue` 中注释写"头指针加1"，实际是尾指针加1；`deQueue` 中注释写"尾指针加1"，实际是头指针加1

4. **内存泄漏**：`createBTNode` 中 `char *str` 通过 `malloc` 分配，但 `main` 函数中从未 `free`。

5. **C/C++ 混用**：同时使用 `new BTNode`（C++）和 `free(BT)`（C），应统一使用 `new/delete` 或 `malloc/free`。参考 [cppreference: operator delete](https://en.cppreference.com/w/cpp/memory/new/operator_delete) — `new` 分配的内存必须用 `delete` 释放，`malloc` 分配的内存用 `free` 释放，混用是未定义行为。

6. **输入缓冲区处理不健壮**：`scanf("%c", &flag)` 后用 `getchar()` 吸收回车，但若用户输入多个字符会导致后续菜单循环异常。

#### 2.2.4 `Tree树/树的增删改查.cpp` — BST 增删改查

**功能概述**：基于 `BstNode` 结构体（`int data`）实现的二叉排序树，包含完整的增删改查、可视化输出。

**具体问题**：

1. **`AddBstNodeToTree` 未处理空树**：函数假设 `root` 不为 NULL，若传入空树会段错误。`CreatBST` 中虽单独处理了空树情况，但 `AddBstNodeToTree` 本身缺少防御性检查。

2. **`DeleteBstNode` 删除根节点逻辑复杂**：使用 `FindFather` 返回父节点，`pos=0` 表示删除根节点，通过 `fap = &(root)` 间接修改根指针。逻辑正确但可读性差，建议封装为更清晰的接口。

3. **`SearchBstNode` 递归版本冗余赋值**：找到节点后执行 `p = T`，实际上直接 `return T` 即可。

4. **`CreateArr` 随机数生成效率低**：使用 O(n²) 的去重方式，当 n 较大时效率低下。建议使用 Fisher-Yates 洗牌算法。

5. **`DrawByLevel` 使用循环队列**：相比 `test.cpp` 的非循环队列，此处使用 `(front+1)%MaxSize` 的循环队列实现，更合理，但未做队列满的判断。

6. **`GetDepth` 计算方式**：空树返回 0，单节点返回 1。与某些教材定义一致，但需在文档中明确约定。

**优点**：BST 删除操作覆盖了三种情况（无子树、单子树、双子树），实现完整；`DrawByTree` 的树形可视化输出实用。

#### 2.2.6 `Tree树/计算二叉树中叶子结点的数目.cpp` — 统计叶子节点

**功能概述**：使用全局变量 `count` 统计二叉树叶子节点数目。

**具体问题**：

#### 2.2.9 `Tree树/text/P5.CPP` — 二叉树操作（含 BinT.h）

**功能概述**：基于 `BinT.h` 头文件的二叉树操作程序，使用 `BiTNode` 结构体（`int data`），包含随机生成二叉树、Map.txt 可视化、递归遍历、叶子统计、深度计算、左右子树交换、BST 操作、层次遍历。

**具体问题**：

1. **`del` 函数使用全局变量 `Parent`**：`search` 函数在遍历时设置全局变量 `Parent`，若 `search` 未找到节点，`Parent` 可能保留上次搜索的值，导致 `del` 中使用错误的父节点。

2. **`del` 函数删除根节点时内存泄漏**：当删除叶子节点且为根节点时，仅设置 `Tree->data = NULLKEY` 而非真正删除节点，后续操作可能访问到标记为 NULLKEY 的"幽灵"节点。

3. **`find` 函数非 BST 查找**：`find` 函数使用普通二叉树的递归查找而非 BST 查找，在 BST 场景下效率为 O(n) 而非 O(log n)。

4. **`InsertBST` 允许重复值**：`data >= (*bst)->data` 时插入右子树，允许重复值存在，可能导致 BST 语义不明确。

5. **`findMin` 使用全局变量 `TempInt`**：与 `Parent` 类似，全局变量在多次调用间可能产生副作用。

6. **`Level_order` 使用 `std::queue`**：与项目其他文件使用自定义队列的风格不一致。

## 5. 新增需求

### 5.1 非递归遍历（优先级：高）

**需求描述**：使用栈实现先序、中序、后序的非递归遍历。

**实现要求**：

- 先序非递归：使用栈，访问根节点 → 右子树入栈 → 左子树入栈
- 中序非递归：使用栈，一路向左入栈 → 弹出访问 → 转向右子树
- 后序非递归：使用栈 + 上次访问标记，或使用双栈法（先序逆序法）
- 栈实现：可使用 C++ STL `std::stack` 或自定义数组栈

**验收标准**：
- 对同一棵二叉树，递归与非递归遍历结果完全一致
- 处理空树、单节点、左斜树、右斜树、完全二叉树等边界情况
- 时间复杂度 O(n)，空间复杂度 O(h)，h 为树高

### 5.2 AVL 树（优先级：高）

**需求描述**：实现自平衡二叉搜索树（AVL 树），支持插入、删除、查找操作。

**实现要求**：

- 节点结构：`data`、`left`、`right`、`height`（或平衡因子 `bf`）
- 四种旋转：LL（右旋）、RR（左旋）、LR（先左旋再右旋）、RL（先右旋再左旋）
- 插入操作：插入后沿路径回溯调整平衡
- 删除操作：删除后沿路径回溯调整平衡
- 查找操作：与 BST 查找一致
- 遍历操作：先序/中序/后序/层次
- 可视化：输出树形结构，标注平衡因子

**验收标准**：
- 插入/删除后树保持平衡（任意节点左右子树高度差 ≤ 1）
- 查找时间复杂度 O(log n)
- 能正确处理引发连续旋转的边界用例（如删除引发双旋转）
- 提供与 BST 对比的性能测试（有序插入场景）

**技术参考**：
- [CLRS](https://mitpress.mit.edu/9780262046305/introduction-to-algorithms/) 第 12-13 章
- C++ STL `std::map` / `std::set` 底层为红黑树而非 AVL，但 AVL 在查找密集场景更优

### 5.3 红黑树（优先级：高）

**需求描述**：实现红黑树，理解 STL 关联容器的底层原理。

**实现要求**：

- 节点结构：`data`、`left`、right`、`parent`、`color`（RED/BLACK）
- 五条性质：
  1. 每个节点是红色或黑色
  2. 根节点是黑色
  3. 每个叶子节点（NIL）是黑色
  4. 红色节点的两个子节点都是黑色
  5. 从任一节点到其每个叶子的所有路径包含相同数目的黑色节点
- 插入操作：插入红色节点 + 旋转 + 重新着色（3 种 case）
- 删除操作：删除 + 修复（4 种 case）
- 查找操作
- 验证函数：检查红黑树五条性质是否满足

**验收标准**：
- 插入/删除后红黑树五条性质全部满足
- 提供验证函数 `isRBTreeValid()` 自动检查
- 查找/插入/删除时间复杂度 O(log n)
- 与 STL `std::map` 进行正确性对比测试

**技术参考**：
- [cppreference: std::map](https://en.cppreference.com/w/cpp/container/map) — GNU libstdc++ 中 `std::map` 底层为红黑树
- [CLRS](https://mitpress.mit.edu/9780262046305/introduction-to-algorithms/) 第 13 章
- Linux 内核 `rbtree` 实现：[kernel.org rbtree](https://docs.kernel.org/core-api/rbtree.html)（截至 2026 年仍活跃维护）

### 5.4 B 树 / B+ 树（优先级：中）

**需求描述**：实现 B 树和 B+ 树，理解数据库索引原理。

**实现要求**：

- B 树：
  - 可配置阶数 `m`（默认 m=3 即 2-3 树）
  - 节点结构：`keys[]`、`children[]`、`n`（当前关键字数）、`is_leaf`
  - 插入操作：关键字满时分裂
  - 删除操作：关键字不足时合并/借位
  - 查找操作
  - 遍历操作（中序）

- B+ 树：
  - 内部节点仅存关键字，数据仅在叶子节点
  - 叶子节点通过链表连接（支持范围查询）
  - 插入/删除/查找/范围查询

**验收标准**：
- B 树插入/删除后满足 m 阶 B 树定义（关键字数 ⌈m/2⌉-1 ~ m-1，子树数 ⌈m/2⌉ ~ m）
- B+ 树范围查询正确
- 提供与 AVL 树在大数据量下的性能对比
- Python 辅助脚本可视化 B 树结构

**技术参考**：
- [CLRS](https://mitpress.mit.edu/9780262046305/introduction-to-algorithms/) 第 18-19 章
- SQLite B 树实现：[sqlite.org fileformat](https://www.sqlite.org/fileformat2.html)（截至 2026 年仍活跃维护）

### 5.5 哈夫曼树（优先级：中）

**需求描述**：实现哈夫曼树的构建、编码与解码。

**实现要求**：

- 哈夫曼树构建：
  - 输入：字符及其权值（频率）
  - 使用最小堆选择两个最小权值节点合并
  - 递归和非递归两种构建方式
- 哈夫曼编码：
  - 从根到叶子的路径即为编码（左 0 右 1）
  - 计算加权路径长度（WPL）
- 哈夫曼解码：
  - 根据编码字符串还原原始字符
- 压缩率计算：
  - 对比等长编码与哈夫曼编码的比特数

**验收标准**：
- 构建的哈夫曼树 WPL 最小
- 编码为前缀编码（无二义性）
- 编码后解码能还原原始数据
- 递归与非递归构建结果一致

**技术参考**：
- [CLRS](https://mitpress.mit.edu/9780262046305/introduction-to-algorithms/) 第 16.3 节
- C++ STL `std::priority_queue` 可用于最小堆实现

### 5.6 字典树（Trie）（优先级：中）

**需求描述**：实现字典树，支持字符串的插入、查找、前缀匹配。

**实现要求**：

- 节点结构：`children[26]`（或 `unordered_map<char, TrieNode*>`）、`is_end`
- 插入操作
- 查找操作（精确匹配）
- 前缀搜索（返回所有以给定前缀开头的单词）
- 删除操作
- 统计操作：单词总数、以某前缀开头的单词数

**进阶需求**：
- 压缩字典树（Radix Tree / Patricia Trie）
- 后缀树（Suffix Tree）基础概念

**验收标准**：
- 插入/查找/删除操作正确
- 前缀搜索返回完整结果集
- 空间效率优于直接存储所有字符串
- 提供 LeetCode 208（Implement Trie）、211（Design Add and Search Words Data Structure）的解题验证

**技术参考**：
- [CLRS](https://mitpress.mit.edu/9780262046305/introduction-to-algorithms/) 习题
- C++ STL `std::unordered_map` 可用于动态子节点映射

### 5.7 线段树（优先级：中）

**需求描述**：实现线段树，支持区间查询与区间更新。

**实现要求**：

- 数组存储方式（完全二叉树，4n 空间）
- 单点更新 + 区间查询
- 区间更新 + 区间查询（懒标记 Lazy Propagation）
- 支持操作：区间求和、区间最大值、区间最小值
- Python 辅助可视化线段树结构

**验收标准**：
- 单点更新/区间查询时间复杂度 O(log n)
- 懒标记区间更新时间复杂度 O(log n)
- 正确处理边界情况（单元素区间、全区间更新）
- 与暴力法对比验证正确性

**技术参考**：
- [CLRS](https://mitpress.mit.edu/9780262046305/introduction-to-algorithms/) 第 14 章（扩张数据结构）
- 竞赛编程经典参考

### 5.8 并查集（优先级：中）

**需求描述**：实现并查集，支持合并与查找操作。

**实现要求**：

- 基础实现：`parent[]` 数组
- 路径压缩（Find 时压缩）
- 按秩合并（Union 时合并）
- 两者结合（近乎 O(1) 的均摊复杂度）
- 应用：连通分量计数、Kruskal 最小生成树

**验收标准**：
- Find + 路径压缩 + 按秩合并的均摊时间复杂度 O(α(n))，α 为反阿克曼函数
- 正确统计连通分量数
- 提供 LeetCode 200（岛屿数量）、547（省份数量）的解题验证
