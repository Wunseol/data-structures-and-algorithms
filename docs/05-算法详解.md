### 插入排序

#### 概念说明

插入排序的基本思想是将待排序序列分为已排序和未排序两部分，每次从未排序部分取出一个元素，在已排序部分中找到合适的位置插入。项目实现了两种变体：直接插入排序和折半插入排序。

#### 权威参考来源

- 《算法导论》（Introduction to Algorithms, CLRS）第 2 章 "Getting Started"，插入排序作为分治思想的入门示例
- Donald E. Knuth, *The Art of Computer Programming*, Vol.3: Sorting and Searching, 第 5.2.1 节，详细分析了插入排序的比较与移动次数

#### 学习前置知识

- 数组的基本操作
- 顺序查找与折半查找
- 时间复杂度 O 符号表示法

#### 实现原理

- **直接插入排序**：从第 2 个元素开始，依次将每个元素插入到前面已排序序列的正确位置。使用顺序查找确定插入位置，同时后移大于待插入元素的记录。
- **折半插入排序**：在直接插入排序的基础上，利用折半查找确定插入位置，减少比较次数，但移动次数不变。

#### 核心代码片段

👉 [直接插入排序完整源码](../src/sorting/sequential_list_sorting.cpp)

```cpp
void InsertSort(SqList &L)
{
	int i,j;
	for(i=2;i<=L.length ;i++)
	{
		if(L.r[i].key <L.r[i-1].key )
		{
			L.r[0]=L.r[i];
			for(j=i-1;L.r[j].key >L.r[0].key ;j--)
			{
				L.r[j+1]=L.r[j];
			}
			L.r[j+1]=L.r [0];
		}
	}
}
```

👉 [折半插入排序完整源码](../src/sorting/sequential_list_sorting.cpp)

```cpp
void BInsertSort(SqList &L)
{
	int i,j,low,mid,high;
	for(i=2;i<=L.length ;i++)
	{
		if(L.r[i].key <L.r[i-1].key )
		{
			L.r[0]=L.r[i];
			low=1;
			high=i-1;
			while(low<=high)
			{
				mid=(low+high)/2;
				if(L.r[0].key <L.r[mid].key ) high=mid-1;
				else low=mid+1;
			}
			for(j=i-1 ;j>=high+1;j--)
			{
				L.r [j+1]=L.r[j];
			}
			L.r[high+1]=L.r[0];
		}
	}
}
```

#### 时间/空间复杂度分析

| 指标 | 直接插入排序 | 折半插入排序 |
|------|-------------|-------------|
| 平均时间复杂度 | O(n²) | O(n²) |
| 最坏时间复杂度 | O(n²) | O(n²) |
| 最好时间复杂度 | O(n) | O(n) |
| 空间复杂度 | O(1) | O(1) |
| 稳定性 | 稳定 | 稳定 |

折半插入排序的比较次数为 O(n log n)，但移动次数仍为 O(n²)，因此总体时间复杂度仍为 O(n²)。

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 纯插入排序 | `std::sort` 使用 IntroSort（快排+堆排+插入排序混合），当子数组长度 ≤ 16 时切换到插入排序 |
| 数据结构 | 顺序表（1-indexed，带哨兵 `r[0]`） | 随机访问迭代器（0-indexed） |
| 哨兵机制 | 使用 `r[0]` 作为哨兵，避免内循环越界判断 | 无哨兵，依赖迭代器比较 |
| 适用场景 | 教学演示 | 工程生产：`std::sort` 在小规模子数组上使用插入排序作为优化 |

#### 已知变体和演进

- **二路插入排序**：利用循环缓冲区减少移动次数
- **表插入排序**：使用链表结构，移动次数降为 0，但比较次数不变
- **希尔排序**：插入排序的分组改进版（见下文）
- **库排序（Library Sort）**：GAP 插入排序，在数组中预留空位，时间复杂度 O(n log n)

#### 版本/平台相关注意事项

- 本项目使用 1-indexed 顺序表（`SqList`），`r[0]` 作为哨兵单元，不存储有效数据
- C++ 标准未规定 `std::sort` 的具体算法，但主流实现（GCC libstdc++、LLVM libc++、MSVC STL）均采用 IntroSort
- 插入排序在近乎有序的数据上接近 O(n)，常被用作高级排序的子过程

#### Python 实现

👉 [Python 实现](../python/algorithms/sorting/insertion_sort.py)

```python
def insertion_sort(arr: list[int]) -> list[int]:
    a = arr[:]
    n = len(a)
    for i in range(1, n):
        if a[i] < a[i - 1]:
            temp = a[i]
            j = i - 1
            while j >= 0 and a[j] > temp:
                a[j + 1] = a[j]
                j -= 1
            a[j + 1] = temp
    return a
```

**与 C/C++ 实现差异**：Python 版本使用 0-indexed 列表和切片复制，无需哨兵单元，函数返回新列表而非原地修改。

### 希尔排序

#### 概念说明

希尔排序又称"缩小增量排序"，是直接插入排序的改进版。通过设置增量序列，将序列分成若干子序列分别进行插入排序，随着增量逐步缩小，序列趋于有序，最终增量为 1 时做一次直接插入排序即可。

#### 权威参考来源

- Donald L. Shell, "A High-Speed Sorting Procedure", *Communications of the ACM*, Vol.2, No.7, 1959
- Knuth, *TAOCP* Vol.3, 第 5.2.1 节，分析了不同增量序列的性能
- Sedgewick, R., "Analysis of Shellsort and Related Algorithms", *ESA*, 1996 — 提出了目前实践中最优的 Sedgewick 增量序列

#### 学习前置知识

- 直接插入排序
- 增量/步长的概念
- 序列渐进有序性

#### 实现原理

1. 选择一个增量序列（本项目使用 {5, 3, 1}）
2. 对每个增量 dk，将序列按 dk 间隔分组，每组内进行插入排序
3. 增量逐步缩小至 1，最终完成排序

#### 核心代码片段

👉 [完整源码](../src/sorting/sequential_list_sorting.cpp)

```cpp
void ShellInsert(SqList &L,int dk)
{
	int i,j;
	for(i=1+dk;i<=L.length ;i++)
	{
		if(L.r[i].key <L.r[i-dk].key )
		{
			L.r[0]=L.r[i];
			for(j=i-dk;j>0 && L.r[j].key >L.r [0].key ;j=j-dk)
			{
				L.r[j+dk]=L.r[j];
			}
			L.r[j+dk]=L.r[0];
		}
	}
}

void ShellSort(SqList &L,int dt[],int t)
{
	int i;
	for(i=0;i<t;i++)
	{
		ShellInsert(L,dt[i]);
	}
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 平均时间复杂度 | O(n^1.3)（依赖增量序列） |
| 最坏时间复杂度 | O(n²)（Shell 原始增量序列） |
| 最好时间复杂度 | O(n) |
| 空间复杂度 | O(1) |
| 稳定性 | 不稳定 |

增量序列对性能影响极大：
- Shell 原始序列 {1,2,4,8,...}：最坏 O(n²)
- Hibbard 序列 {1,3,7,15,...}：最坏 O(n^1.5)
- Sedgewick 序列 {1,5,19,41,109,...}：最坏 O(n^1.33)

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 希尔排序 | `std::sort` 不使用希尔排序，采用 IntroSort |
| 增量序列 | 硬编码 {5,3,1} | 不适用 |
| 实际地位 | 教学演示 | 希尔排序在现代 C++ 标准库中未被采用，因 IntroSort 在最坏情况下更优 |

#### 已知变体和演进

- **Hibbard 增量**：2^k - 1，最坏 O(n^1.5)
- **Sedgewick 增量**：4^k + 3·2^(k-1) + 1，最坏 O(n^1.33)，实践中最优
- **Pratt 增量**：3-smooth 数列，最坏 O(n log²n) 但常数较大
- **Ciura 增量**：经验最优序列 {1,4,10,23,57,132,301,701}，无严格理论证明

#### 版本/平台相关注意事项

- 本项目增量序列为硬编码，实际应用中应根据数据规模动态选择
- 希尔排序不稳定：相同关键字的记录可能因分组不同而改变相对顺序
- C++ 标准库未提供希尔排序接口

#### Python 实现

👉 [Python 实现](../python/algorithms/sorting/shell_sort.py)

```python
def shell_sort(arr: list[int], gaps: list[int] | None = None) -> list[int]:
    a = arr[:]
    n = len(a)
    if gaps is None:
        gaps = []
        g = n // 2
        while g > 0:
            gaps.append(g)
            g //= 2
    for dk in gaps:
        for i in range(dk, n):
            if a[i] < a[i - dk]:
                temp = a[i]
                j = i - dk
                while j >= 0 and a[j] > temp:
                    a[j + dk] = a[j]
                    j -= dk
                a[j + dk] = temp
    return a
```

**与 C/C++ 实现差异**：Python 版本默认使用 Shell 增量序列（n//2 递减），支持自定义增量序列参数，无需硬编码增量数组。

### 冒泡排序

#### 概念说明

冒泡排序通过相邻元素的比较和交换，使较大元素逐渐"冒泡"到序列末端。每趟排序将当前未排序部分的最大元素放到正确位置。

#### 权威参考来源

- 《算法导论》第 2 章，练习题 2-2 详细分析了冒泡排序的正确性证明
- Knuth, *TAOCP* Vol.3, 第 5.2.2 节，分析了相邻交换排序

#### 学习前置知识

- 数组的基本操作
- 交换操作
- 不变量与循环不变式的概念

#### 实现原理

1. 从前往后依次比较相邻元素，若逆序则交换
2. 每趟排序后，最大元素到达末端
3. 设置 flag 标志，若某趟无交换则提前结束

#### 核心代码片段

👉 [完整源码](../src/sorting/sequential_list_sorting.cpp)

```cpp
void BubbleSort(SqList &L)
{
	int i,j,flag=1;
	RedType t;
	for(i=L.length -1;i>=1&&flag;i--)
	{
		flag=0;
		for(j=1;j<=i;j++)
		{
			if(L.r [j].key >L.r[j+1].key )
			{
				flag=1;
				t=L.r [j];
				L.r[j]=L.r[j+1];
				L.r[j+1]=t;
			}
		}
	}
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 平均时间复杂度 | O(n²) |
| 最坏时间复杂度 | O(n²) |
| 最好时间复杂度 | O(n)（已有序时，flag 优化生效） |
| 空间复杂度 | O(1) |
| 稳定性 | 稳定 |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 冒泡排序 | `std::sort` 不使用冒泡排序 |
| 交换次数 | O(n²) | IntroSort 交换次数远少于冒泡排序 |
| 实际地位 | 教学演示 | 冒泡排序在实际工程中几乎不被使用，性能远逊于 `std::sort` |

#### 已知变体和演进

- **鸡尾酒排序（Cocktail Sort）**：双向冒泡，交替从左到右和从右到左扫描
- **梳排序（Comb Sort）**：用大于 1 的间距替代相邻比较，类似希尔排序对冒泡的改进
- **奇偶排序（Odd-Even Sort）**：交替比较奇数索引对和偶数索引对，适合并行化

#### 版本/平台相关注意事项

- flag 优化使得对已有序序列只需一趟扫描即可终止
- 冒泡排序的常数因子较大，实际性能在 O(n²) 排序中最差
- C++ 标准库未提供冒泡排序接口

#### Python 实现

👉 [Python 实现](../python/algorithms/sorting/bubble_sort.py)

```python
def bubble_sort(arr: list[int]) -> list[int]:
    a = arr[:]
    n = len(a)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if a[j] > a[j + 1]:
                a[j], a[j + 1] = a[j + 1], a[j]
                swapped = True
        if not swapped:
            break
    return a
```

### 快速排序

#### 概念说明

快速排序采用分治策略，选取一个枢轴（pivot）元素，通过一趟划分将序列分为两部分：左边元素均小于枢轴，右边元素均大于等于枢轴，然后递归排序左右子序列。

#### 权威参考来源

- C.A.R. Hoare, "Quicksort", *The Computer Journal*, Vol.5, No.1, 1962 — 快速排序的原始论文
- 《算法导论》第 7 章 "Quicksort"，详细分析了划分策略与随机化
- David R. Musser, "Introspective Sorting and Selection Algorithms", *Software: Practice and Experience*, 1997 — IntroSort 的提出
- GCC libstdc++ 源码：`std::sort` 使用 IntroSort，递归深度超过 2·log₂n 时切换为堆排序

#### 学习前置知识

- 分治策略
- 递归
- 数组划分（Partition）操作
- 栈空间与递归深度的关系

#### 实现原理

1. 选取第一个元素作为枢轴，保存到 `L.r[0]`
2. 从两端向中间扫描，将比枢轴小的移到低端，比枢轴大的移到高端
3. 枢轴归位，递归处理左右子序列

#### 核心代码片段

👉 [完整源码](../src/sorting/sequential_list_sorting.cpp)

```cpp
int Partition(SqList &L,int low,int high)
{
	L.r[0]=L.r[low];
	int pivotkey=L.r[low].key;
	while(low<high)
	{
		while(low<high&&L.r[high].key>=pivotkey) --high;
		L.r[low]=L.r[high];
		while(low<high&&L.r[low].key<=pivotkey) ++low;
		L.r[high]=L.r[low];
	}
	L.r[low]=L.r[0];
	return low;
}

void QSort(SqList &L,int low,int high)
{
	if(low<high)
	{
		int pivotloc=Partition(L,low,high);
		QSort(L,low,pivotloc-1);
		QSort(L,pivotloc+1,high);
	}
}

void QuickSort(SqList &L)
{
	QSort(L,1,L.length);
}
```

| 最坏时间复杂度 | O(n²)（已有序且选首元素为枢轴时） |
| 最好时间复杂度 | O(n log n) |
| 空间复杂度 | O(log n)（递归栈），最坏 O(n) |
| 稳定性 | 不稳定 |

|------|-----------|---------------------|

#### 已知变体和演进

- **随机化快速排序**：随机选择枢轴，期望 O(n log n)
- **三数取中快排**：选首、中、尾三元素的中位数作为枢轴
- **IntroSort**（Musser 1997）：快排+堆排+插入排序混合，C++ STL `std::sort` 的实现基础
- **双轴快速排序（Dual-Pivot Quicksort）**：Vladimir Yaroslavskiy 2009 年提出，Java 7+ 的 `Arrays.sort()` 对基本类型使用此算法

- 本项目实现为经典 Hoare 划分，对已有序序列会退化为 O(n²)

### 选择排序

#### 概念说明

选择排序每趟从未排序部分选出最小元素，与未排序部分的第一个元素交换，逐步扩大已排序区间。

#### 权威参考来源

- 《算法导论》第 2 章，练习题 2.2-2 要求证明选择排序的循环不变式
- Knuth, *TAOCP* Vol.3, 第 5.2.3 节

#### 学习前置知识

- 数组的基本操作
- 最小值查找
- 交换操作

#### 实现原理

1. 第 i 趟排序从 `L.r[i]` 到 `L.r[n]` 中选出关键字最小的记录
2. 将最小记录与第 i 个记录交换
3. 重复 n-1 趟

#### 核心代码片段

👉 [完整源码](../src/sorting/sequential_list_sorting.cpp)

```cpp
void SelectSort(SqList &L)
{
	for(int i=1;i<L.length;++i)
	{
		int k=i;
		for(int j=i+1;j<=L.length;++j)
			if(L.r[j].key<L.r[k].key) k=j;
		if(k!=i)
			{RedType t=L.r[i];L.r[i]=L.r[k];L.r[k]=t;}
	}
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 平均时间复杂度 | O(n²) |
| 最坏时间复杂度 | O(n²) |
| 最好时间复杂度 | O(n²)（即使已有序也需完整比较） |
| 空间复杂度 | O(1) |
| 稳定性 | 不稳定 |

选择排序的比较次数恒为 n(n-1)/2，但交换次数最多 n-1 次，在交换代价大的场景中有优势。

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 简单选择排序 | `std::sort` 使用 IntroSort |
| 选择第 k 小 | 需完整排序 | `std::nth_element` 使用 IntroSelect（BFPRT 选择算法的变体），平均 O(n) |
| 部分排序 | 不支持 | `std::partial_sort` 使用堆排序，O(n log k) |

#### 已知变体和演进

- **锦标赛排序（Tournament Sort）**：利用树形结构减少比较次数，O(n log n)
- **堆排序**：选择排序的改进版，使用堆数据结构高效选取最大/最小值
- **BFPRT 选择算法**（Blum-Floyd-Pratt-Rivest-Tarjan, 1973）：最坏 O(n) 的选择算法，`std::nth_element` 的理论基础

#### 版本/平台相关注意事项

- 选择排序不稳定：交换可能改变相同关键字记录的相对顺序（如序列 5,5,2，第一趟交换后两个 5 的顺序改变）
- `std::nth_element` 在 C++17 后支持并行执行策略

#### Python 实现

👉 [Python 实现](../python/algorithms/sorting/selection_sort.py)

```python
def selection_sort(arr: list[int]) -> list[int]:
    a = arr[:]
    n = len(a)
    for i in range(n - 1):
        k = i
        for j in range(i + 1, n):
            if a[j] < a[k]:
                k = j
        if k != i:
            a[i], a[k] = a[k], a[i]
    return a
```

### 堆排序

#### 概念说明

堆排序利用堆这种数据结构进行排序。将序列建成大根堆后，堆顶元素为最大值，将其与末尾元素交换并缩小堆的范围，再调整堆，重复此过程即可完成排序。

#### 权威参考来源

- J.W.J. Williams, "Algorithm 232: Heapsort", *Communications of the ACM*, 1964 — 堆排序的原始提出
- R.W. Floyd, "Algorithm 245: Treesort 3", *Communications of the ACM*, 1964 — 提出了自底向上的树堆化（heapify）方法，将建堆时间从 O(n log n) 优化到 O(n)
- 《算法导论》第 6 章 "Heapsort"

#### 学习前置知识

- 完全二叉树的性质
- 数组表示完全二叉树（下标关系：父节点 i，左子 2i，右子 2i+1）
- 堆的性质（大根堆/小根堆）

#### 实现原理

1. **建堆**：从最后一个非叶子节点开始，自底向上调用 `HeapAdjust` 将序列建成大根堆
2. **排序**：每次将堆顶（最大值）与堆末元素交换，堆范围减 1，再对堆顶执行下沉调整
3. **调整**：`HeapAdjust` 将以 s 为根的子树调整为大根堆（假设其子树已是大根堆）

#### 核心代码片段

👉 [完整源码](../src/sorting/sequential_list_sorting.cpp)

```cpp
void HeapAdjust(SqList &L,int s,int m)
{
	RedType rc=L.r[s];
	for(int j=2*s;j<=m;j*=2)
	{
		if(j<m&&L.r[j].key<L.r[j+1].key) ++j;
		if(rc.key>=L.r[j].key) break;
		L.r[s]=L.r[j];s=j;
	}
	L.r[s]=rc;
}

void CreatHeap(SqList &L)
{
	int n=L.length;
	for(int i=n/2;i>0;--i)
		HeapAdjust(L,i,n);
}

void HeapSort(SqList &L)
{
	CreatHeap(L);
	for(int i=L.length;i>1;--i)
	{
		RedType x=L.r[1];
		L.r[1]=L.r[i];
		L.r[i]=x;
		HeapAdjust(L,1,i-1);
	}
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 建堆时间 | O(n)（Floyd 方法） |
| 平均时间复杂度 | O(n log n) |
| 最坏时间复杂度 | O(n log n) |
| 最好时间复杂度 | O(n log n) |
| 空间复杂度 | O(1) |
| 稳定性 | 不稳定 |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 堆排序 | `std::sort` 使用 IntroSort（包含堆排序作为退化策略） |
| 堆操作 | 手动实现 | `std::make_heap`、`std::push_heap`、`std::pop_heap`、`std::sort_heap` |
| 部分排序 | 不支持 | `std::partial_sort` 内部使用堆排序，O(n log k) |
| 优先队列 | 不适用 | `std::priority_queue` 底层使用 `std::make_heap` 系列函数 |

#### 已知变体和演进

- **Floyd 建堆法**：自底向上，O(n) 建堆，本项目即采用此法
- **Williams 建堆法**：逐个插入，O(n log n) 建堆
- **Smoothsort**：Edsger Dijkstra 1981 年提出，在已有序数据上接近 O(n)，最坏仍 O(n log n)
- **内省排序（IntroSort）**：快排退化时切换为堆排序，保证 O(n log n) 最坏复杂度

#### 版本/平台相关注意事项

- 本项目使用 1-indexed 数组表示堆，父节点 i 的子节点为 2i 和 2i+1
- C++ STL 的堆操作使用 0-indexed，父节点 i 的子节点为 2i+1 和 2i+2
- `std::priority_queue` 默认为大根堆，小根堆需声明 `std::priority_queue<int, std::vector<int>, std::greater<int>>`
- C++20 新增 `std::make_heap` 等的 constexpr 版本

#### Python 实现

👉 [Python 实现](../python/algorithms/sorting/heap_sort.py)

```python
def _sift_down(arr: list[int], start: int, end: int) -> None:
    root = start
    while True:
        child = 2 * root + 1
        if child > end:
            break
        if child + 1 <= end and arr[child] < arr[child + 1]:
            child += 1
        if arr[root] >= arr[child]:
            break
        arr[root], arr[child] = arr[child], arr[root]
        root = child

def heap_sort(arr: list[int]) -> list[int]:
    a = arr[:]
    n = len(a)
    if n <= 1:
        return a
    for i in range(n // 2 - 1, -1, -1):
        _sift_down(a, i, n - 1)
    for i in range(n - 1, 0, -1):
        a[0], a[i] = a[i], a[0]
        _sift_down(a, 0, i - 1)
    return a
```

**与 C/C++ 实现差异**：Python 版本使用 0-indexed 数组（子节点为 2i+1 和 2i+2），C++ 版本使用 1-indexed（子节点为 2i 和 2i+1）。

### 归并排序

#### 概念说明

归并排序采用分治策略，将序列递归地分成两半分别排序，然后将两个有序子序列归并为一个有序序列。

#### 权威参考来源

- John von Neumann, 1945 年提出归并排序（据 Knuth 记载）
- 《算法导论》第 2 章 "Getting Started" 与第 4 章 "Divide and Conquer"，归并排序作为分治范例
- Knuth, *TAOCP* Vol.3, 第 5.2.4 节，详细分析了归并排序的各种变体

#### 学习前置知识

- 分治策略
- 递归
- 双指针合并两个有序序列

#### 实现原理

1. **分解**：将序列递归地分为左右两半
2. **求解**：递归排序左右子序列
3. **合并**：将两个有序子序列归并，使用双指针依次比较取较小者

#### 核心代码片段

👉 [完整源码](../src/sorting/sequential_list_sorting.cpp)

```cpp
void Merge(RedType R[],RedType T[],int low,int mid,int high)
{
	int i=low,j=mid+1,k=low;
	while(i<=mid&&j<=high)
	{
		if(R[i].key<=R[j].key) T[k++]=R[i++];
		else T[k++]=R[j++];
	}
	while(i<=mid) T[k++]=R[i++];
	while(j<=high) T[k++]=R[j++];
}

void MSort(RedType R[],RedType T[],int low,int high)
{
	int mid;
	RedType *TR2=new RedType[high-low+1];
	if(low==high) T[low]=R[low];
	else{
		mid=(low+high)/2;
		MSort(R,TR2,low,mid);
		MSort(R,TR2,mid+1,high);
		Merge(TR2,T,low,mid,high);
	}
	delete[] TR2;
}

void MergeSort(SqList &L)
{
	MSort(L.r,L.r,1,L.length);
}
```

#### Python 实现

👉 [Python 实现](../python/algorithms/sorting/merge_sort.py)

```python
def _merge(left: list[int], right: list[int]) -> list[int]:
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

def merge_sort(arr: list[int]) -> list[int]:
    if len(arr) <= 1:
        return arr[:]
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return _merge(left, right)
```

- 稳定排序的定义

#### 实现原理

- **基本版**：统计每个值的出现次数，然后按次数输出
- **稳定版**：在计数数组上做前缀和，然后从后往前遍历原数组，利用前缀和确定每个元素的最终位置，保证稳定性

#### 核心代码片段

👉 [完整源码](../src/sorting/counting_sort.cpp)

```cpp
vector<int> countingSortStable(const vector<int>& arr) {
    if (arr.empty()) return {};

    int maxVal = *max_element(arr.begin(), arr.end());
    int minVal = *min_element(arr.begin(), arr.end());
    int range = maxVal - minVal + 1;

    vector<int> count(range, 0);
    for (int x : arr) {
        count[x - minVal]++;
    }

    for (int i = 1; i < range; i++) {
        count[i] += count[i - 1];
    }

    vector<int> output(arr.size());
    for (int i = static_cast<int>(arr.size()) - 1; i >= 0; i--) {
        output[count[arr[i] - minVal] - 1] = arr[i];
        count[arr[i] - minVal]--;
    }
    return output;
}
```

|------|-----|
| 平均时间复杂度 | O(n+k) |
| 最坏时间复杂度 | O(n+k) |
| 空间复杂度 | O(n+k) |
| 稳定性 | 稳定（稳定版） |

其中 k 为数据范围（maxVal - minVal + 1）。

| 适用类型 | 整数 | `std::sort` 适用于任意可比较类型 |

```python
        return []
    max_val = max(arr)
    min_val = min(arr)
    range_size = max_val - min_val + 1

    for i in range(1, range_size):
        count[i] += count[i - 1]
    output = [0] * len(arr)
    for i in range(len(arr) - 1, -1, -1):
        output[count[arr[i] - min_val] - 1] = arr[i]
        count[arr[i] - min_val] -= 1
```

### 基数排序

#### 概念说明

基数排序按关键字的各位（从低位到高位，LSD 方式）进行排序，每一轮使用稳定的计数排序对某一位进行排序，最终得到有序序列。

#### 权威参考来源

- 《算法导论》第 8 章，8.3 节 "Radix Sort"
- H.S. Bright, "Sorting with Large Storage", 1964 — LSD 基数排序的系统化描述

#### 学习前置知识

- 计数排序（作为基数排序的稳定子过程）
- 数字的位表示
- 稳定排序的必要性

#### 实现原理

1. 找出最大值，确定位数 d
2. 从最低位到最高位，对每一位执行计数排序
3. 每轮排序保持前一轮的相对顺序（稳定性保证正确性）

#### 核心代码片段

👉 [完整源码](../src/sorting/radix_sort.cpp)

```cpp
void countingSortByDigit(vector<int>& arr, int exp) {
    int n = static_cast<int>(arr.size());
    vector<int> output(n);
    vector<int> count(10, 0);

    for (int i = 0; i < n; i++) {
        count[(arr[i] / exp) % 10]++;
    }

    for (int i = 1; i < 10; i++) {
        count[i] += count[i - 1];
    }

    for (int i = n - 1; i >= 0; i--) {
        int digit = (arr[i] / exp) % 10;
        output[count[digit] - 1] = arr[i];
        count[digit]--;
    }

    arr = output;
}

void radixSort(vector<int>& arr) {
    if (arr.empty()) return;

    int maxVal = *max_element(arr.begin(), arr.end());

    for (int exp = 1; maxVal / exp > 0; exp *= 10) {
        countingSortByDigit(arr, exp);
    }
}
```

| 稳定性 | 稳定 |

其中 d 为最大位数，k 为基数（本项目为 10）。

### 桶排序

#### 概念说明

桶排序将数据按范围分到若干个桶中，每个桶内部使用插入排序，最后按桶顺序合并结果。适用于数据均匀分布的场景。

#### 权威参考来源

- 《算法导论》第 8 章，8.4 节 "Bucket Sort"
- 桶排序的平均 O(n) 分析基于输入均匀分布的假设

#### 学习前置知识

- 插入排序（桶内排序）
- 哈希映射（元素到桶的映射）
- 概率分析基础

#### 实现原理

1. 创建 n 个桶，每个桶对应一个值域区间
2. 将每个元素按值映射到对应桶中
3. 对每个桶内部使用插入排序
4. 按桶顺序依次输出所有元素

#### 核心代码片段

👉 [完整源码](../src/sorting/bucket_sort.cpp)

```cpp
void bucketSort(vector<double>& arr) {
    int n = static_cast<int>(arr.size());
    if (n <= 1) return;

    vector<vector<double>> buckets(n);

    for (int i = 0; i < n; i++) {
        int bucketIndex = static_cast<int>(n * arr[i]);
        if (bucketIndex >= n) bucketIndex = n - 1;
        buckets[bucketIndex].push_back(arr[i]);
    }

    for (int i = 0; i < n; i++) {
        insertionSort(buckets[i]);
    }

    int index = 0;
    for (int i = 0; i < n; i++) {
        for (double val : buckets[i]) {
            arr[index++] = val;
        }
    }
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 平均时间复杂度 | O(n)（数据均匀分布时） |
| 最坏时间复杂度 | O(n²)（所有元素落入同一桶） |
| 空间复杂度 | O(n+k) |
| 稳定性 | 稳定（桶内排序稳定时） |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 桶排序 | C++ STL 未提供桶排序接口 |
| 数据范围 | [0, 1) 浮点数 | `std::sort` 适用于任意可比较类型 |
| 桶内排序 | 插入排序 | 不适用 |

#### 已知变体和演进

- **桶内排序选择**：桶内元素多时可改用快排或归并排序
- **动态桶数量**：根据数据分布自适应调整桶数量和范围
- **并行桶排序**：各桶独立排序，天然适合并行化

#### 版本/平台相关注意事项

- 本项目假设输入为 [0, 1) 区间的浮点数，其他范围需调整映射函数

- C++ STL 未提供桶排序接口

👉 [Python 实现](../python/algorithms/sorting/bucket_sort.py)

```python
def bucket_sort(arr: list[float]) -> list[float]:
    n = len(arr)
    if n <= 1:
        return arr[:]
    buckets: list[list[float]] = [[] for _ in range(n)]
    for x in arr:
        bucket_index = int(n * x)
        if bucket_index >= n:
            bucket_index = n - 1
        buckets[bucket_index].append(x)
    for i in range(n):
        buckets[i] = _insertion_sort(buckets[i])
    result: list[float] = []
    for bucket in buckets:
        result.extend(bucket)
    return result
```

### 二分查找

#### 概念说明

二分查找在有序数组中通过不断将查找区间减半来定位目标值。项目还实现了 lowerBound、upperBound 以及旋转数组查找等变体。

#### 权威参考来源

- 《算法导论》第 2 章，二分查找作为分治的典型应用
- Jon Bentley, *Programming Pearls*, 第 2 章与第 4 章，详细讨论了二分查找的正确实现与常见错误
- Joshua Bloch, "Extra, Extra - Read All About It: Nearly All Binary Searches and Mergesorts are Broken", 2006 — 指出 `mid = (low + high) / 2` 的整数溢出问题

#### 学习前置知识

- 有序数组
- 区间表示法（闭区间 vs 半开区间）
- 整数溢出

#### 实现原理

1. 设定查找区间 [low, high]
2. 取中间位置 mid = low + (high - low) / 2，避免溢出
3. 比较 arr[mid] 与 target，根据比较结果缩小查找区间
4. 重复直到找到目标或区间为空

#### 核心代码片段

👉 [完整源码](../src/searching/binary_search.cpp)

```cpp
SearchResult binarySearch(const vector<int>& arr, int target) {
    SearchResult result;
    int low = 0, high = static_cast<int>(arr.size()) - 1;

    while (low <= high) {
        int mid = low + (high - low) / 2;
        result.comparisons++;
        if (arr[mid] == target) {
            result.found = true;
            result.index = mid;
            return result;
        } else if (arr[mid] < target) {
            low = mid + 1;
        } else {
            high = mid - 1;
        }
    }
    return result;
}
```

|------|-----|
| 平均时间复杂度 | O(log n) |
| 最坏时间复杂度 | O(log n) |
| 最好时间复杂度 | O(1) |
| 空间复杂度 | O(1) |

#### 与 C++ STL 标准库实现的对比

| 下界查找 | 不支持 | `std::lower_bound` 返回第一个 ≥ target 的位置 |

### 插值查找

#### 概念说明

插值查找是二分查找的改进，根据目标值在当前区间的相对位置来估计其索引，而非简单取中点。适用于数据分布均匀的有序数组。

#### 权威参考来源

- 《算法导论》习题 11.2-6 讨论了插值查找
- Weiss, *Data Structures and Algorithm Analysis in C++*, 第 2.6 节
- Perl, Itai, Avni, "Interpolation Search—A Log Log N Search", *Communications of the ACM*, 1978

#### 学习前置知识

- 二分查找
- 线性插值
- 数据分布的概念

#### 实现原理

插值公式：`pos = low + (target - arr[low]) / (arr[high] - arr[low]) * (high - low)`

当数据均匀分布时，该公式能更精确地估计目标位置，减少比较次数。

#### 核心代码片段

👉 [完整源码](../src/searching/interpolation_search.cpp)

```cpp
SearchResult interpolationSearch(const vector<int>& arr, int target) {
    SearchResult result;
    int low = 0, high = static_cast<int>(arr.size()) - 1;

    while (low <= high && target >= arr[low] && target <= arr[high]) {
        result.comparisons++;
        if (arr[high] == arr[low]) {
            if (arr[low] == target) {
                result.found = true;
                result.index = low;
            }
            return result;
        }

        int pos = low + static_cast<int>(
            (static_cast<double>(target - arr[low]) / (arr[high] - arr[low])) * (high - low)
        );

        if (pos < low) pos = low;
        if (pos > high) pos = high;

        result.comparisons++;
        if (arr[pos] == target) {
            result.found = true;
            result.index = pos;
            return result;
        } else if (arr[pos] < target) {
            low = pos + 1;
        } else {
            high = pos - 1;
        }
    }
    return result;
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 平均时间复杂度 | O(log log n)（均匀分布时） |
| 最坏时间复杂度 | O(n) |
| 空间复杂度 | O(1) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 插值查找 | STL 未提供插值查找 |
| 假设 | 数据均匀分布 | `std::lower_bound` 不假设分布 |
| 实际地位 | 特定场景优化 | STL 二分查找通用性更强 |

#### 已知变体和演进

- **插值-二分混合查找**：先用插值估计，不收敛时切换为二分
- **自适应插值查找**：Andersson 与 Petersson 1995 年提出，O(log log n) 最坏保证

#### 版本/平台相关注意事项

- 插值公式中除法需注意 arr[high] == arr[low] 的边界
- 本项目使用浮点运算计算插值位置，整数运算版本需注意精度
- 对非均匀分布数据（如指数分布），性能可能不如二分查找

#### Python 实现

👉 [Python 实现](../python/algorithms/searching/interpolation_search.py)

```python

    low, high = 0, len(arr) - 1
    while low <= high and target >= arr[low] and target <= arr[high]:
        if arr[high] == arr[low]:
            if arr[low] == target:
                return low
            return -1
        pos = low + int((target - arr[low]) / (arr[high] - arr[low]) * (high - low))
        if pos < low:
            pos = low
        if pos > high:
            pos = high
        if arr[pos] == target:
            return pos
        elif arr[pos] < target:
            low = pos + 1
        else:
            high = pos - 1
    return -1
```

**与 C/C++ 实现差异**：Python 版本使用浮点除法计算插值位置，无需 static_cast 类型转换。

### 斐波那契查找

#### 概念说明

斐波那契查找利用斐波那契数列来分割查找区间，是二分查找的另一种变体。它只使用加减法运算，避免了除法开销。

#### 权威参考来源

- 《数据结构（C语言版）》（严蔚敏）详细介绍了斐波那契查找
- Fibonacci 查找的分割比例接近黄金分割比 0.618

#### 学习前置知识

- 二分查找
- 斐波那契数列
- 黄金分割比

#### 实现原理

1. 找到大于等于 n 的最小斐波那契数 fibM
2. 用 fibM 的前两个斐波那契数 fib1、fib2 来确定分割点
3. 比较分割点与目标值，根据结果调整斐波那契数和偏移量
4. 重复直到找到目标或区间为空

#### 核心代码片段

👉 [完整源码](../src/searching/fibonacci_search.cpp)

```cpp
SearchResult fibonacciSearch(const vector<int>& arr, int target) {
    SearchResult result;
    int n = static_cast<int>(arr.size());
    if (n == 0) return result;

    int fib2 = 0;
    int fib1 = 1;
    int fibM = fib2 + fib1;

    while (fibM < n) {
        fib2 = fib1;
        fib1 = fibM;
        fibM = fib2 + fib1;
    }

    int offset = -1;

    while (fibM > 1) {
        int i = min(offset + fib2, n - 1);

        result.comparisons++;
        if (arr[i] < target) {
            fibM = fib1;
            fib1 = fib2;
            fib2 = fibM - fib1;
            offset = i;
        } else if (arr[i] > target) {
            result.comparisons++;
            fibM = fib2;
            fib1 = fib1 - fib2;
            fib2 = fibM - fib1;
        } else {
            result.comparisons++;
            result.found = true;
            result.index = i;
            return result;
        }
    }

    if (fib1 == 1 && offset + 1 < n) {
        result.comparisons++;
        if (arr[offset + 1] == target) {
            result.found = true;
            result.index = offset + 1;
        }
    }
    return result;
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 平均时间复杂度 | O(log n) |
| 最坏时间复杂度 | O(log n) |
| 空间复杂度 | O(1) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 斐波那契查找 | STL 未提供斐波那契查找 |
| 运算 | 仅加减法 | `std::lower_bound` 使用除法计算 mid |
| 实际地位 | 教学演示 | 现代处理器上除法开销可忽略，斐波那契查找的实际优势不大 |

#### 已知变体和演进

- **黄金分割搜索**：一维连续函数求极值的类似方法
- **与插值查找结合**：利用斐波那契分割的非对称性适应特定分布

#### Python 实现

👉 [Python 实现](../python/algorithms/searching/fibonacci_search.py)

```python
def fibonacci_search(arr: list[int], target: int) -> int:
    n = len(arr)
    if n == 0:
        return -1
    fib2, fib1 = 0, 1
    fib_m = fib2 + fib1
    while fib_m < n:
        fib2, fib1 = fib1, fib_m
        fib_m = fib2 + fib1
    offset = -1
    while fib_m > 1:
        i = min(offset + fib2, n - 1)
        if arr[i] < target:
            fib_m, fib1, fib2 = fib1, fib2, fib_m - fib1
            offset = i
        elif arr[i] > target:
            fib_m, fib1, fib2 = fib2, fib1 - fib2, fib_m - fib1
        else:
            return i
    if fib1 == 1 and offset + 1 < n and arr[offset + 1] == target:
        return offset + 1
    return -1
```

## 字符串算法

### 暴力匹配

#### 概念说明

暴力匹配（Brute Force）是最朴素的字符串匹配算法，从主串的每个位置开始，逐个字符与模式串比较。

#### 权威参考来源

- 《算法导论》第 32 章 "String Matching"，暴力匹配作为基准算法
- 所有高级字符串匹配算法均以暴力匹配为起点进行优化

#### 学习前置知识

- 字符串的基本操作
- 双重循环

#### 实现原理

1. 对主串的每个起始位置 i（0 到 n-m），逐个比较模式串字符
2. 若全部匹配则记录位置；若某字符不匹配则 i 后移一位重新开始

#### 核心代码片段

👉 [完整源码](../src/string_algorithm/brute_force_match.cpp)

```cpp
std::vector<MatchResult> bruteForceMatch(const std::string& text, const std::string& pattern) {
    std::vector<MatchResult> results;
    int n = text.length();
    int m = pattern.length();
    int comparisons = 0;

    for (int i = 0; i <= n - m; i++) {
        int j = 0;
        while (j < m) {
            comparisons++;
            if (text[i + j] != pattern[j]) {
                break;
            }
            j++;
        }
        if (j == m) {
            results.push_back({i, comparisons});
        }
    }

    if (results.empty()) {
        results.push_back({-1, comparisons});
    }
    return results;
}
```

| 指标 | 值 |
|------|-----|
| 平均时间复杂度 | O(n+m) |
| 最坏时间复杂度 | O(nm) |
| 空间复杂度 | O(1) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 暴力匹配 | `std::string::find` 实现未规定，主流实现使用优化算法 |
| 返回值 | 所有匹配位置 | 第一个匹配位置 |
| 性能 | 最坏 O(nm) | 主流实现通常优于暴力匹配 |

#### 已知变体和演进

- 暴力匹配是所有字符串匹配算法的基准

### KMP 算法

#### 概念说明

KMP（Knuth-Morris-Pratt）算法通过预处理模式串构建 next/nextval 数组，在匹配失败时利用已匹配信息跳过不必要的比较，避免主串指针回退。

#### 权威参考来源

- Donald E. Knuth, James H. Morris, Jr., Vaughan R. Pratt, "Fast Pattern Matching in Strings", *SIAM Journal on Computing*, Vol.6, No.2, 1977 — KMP 算法的原始论文
- 《算法导论》第 32 章，32.4 节 "The Knuth-Morris-Pratt algorithm"

#### 学习前置知识

- 暴力字符串匹配
- 前缀函数/部分匹配表
- 自动机的概念

#### 实现原理

1. **构建 next 数组**：next[i] 表示模式串前 i 个字符中，前缀与后缀的最长公共长度
2. **构建 nextval 数组**：对 next 数组的优化，当 `pattern[i] == pattern[next[i]]` 时直接跳到 `nextval[next[i]]`
3. **匹配过程**：主串指针不回退，失配时根据 next/nextval 数组调整模式串指针

#### 核心代码片段

👉 [完整源码](../src/string_algorithm/kmp.cpp)

```cpp
std::vector<int> computeNext(const std::string& pattern) {
    int m = pattern.length();
    std::vector<int> next(m, 0);
    next[0] = -1;
    int i = 0, j = -1;

    while (i < m - 1) {
        if (j == -1 || pattern[i] == pattern[j]) {
            i++;
            j++;
            next[i] = j;
        } else {
            j = next[j];
        }
    }
    return next;
}

std::vector<int> computeNextval(const std::string& pattern) {
    int m = pattern.length();
    std::vector<int> nextval(m, 0);
    nextval[0] = -1;
    int i = 0, j = -1;

    while (i < m - 1) {
        if (j == -1 || pattern[i] == pattern[j]) {
            i++;
            j++;
            if (pattern[i] != pattern[j]) {
                nextval[i] = j;
            } else {
                nextval[i] = nextval[j];
            }
        } else {
            j = nextval[j];
        }
    }
    return nextval;
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 预处理时间复杂度 | O(m) |
| 匹配时间复杂度 | O(n) |
| 总时间复杂度 | O(n+m) |
| 空间复杂度 | O(m) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | KMP | `std::string::find` 不保证使用 KMP |
| 预处理 | next/nextval 数组 | STL 实现可能使用其他算法 |
| 多模式匹配 | 不支持 | 需使用 Aho-Corasick 自动机 |

#### 已知变体和演进

- **nextval 优化**：本项目实现，避免 `pattern[i] == pattern[next[i]]` 时的冗余比较
- **KMP 自动机**：将 next 数组扩展为完整的有限状态自动机
- **Aho-Corasick 算法**（1975）：KMP 的多模式扩展，构建 Trie + 失败指针
- **Z 算法（Z-function）**：另一种 O(n+m) 字符串匹配方法，某些场景下比 KMP 更直观

#### 版本/平台相关注意事项

- next 数组的不同定义方式（0-indexed vs 1-indexed，-1 开头 vs 0 开头）容易导致混淆
- 本项目 next[0] = -1，部分教材使用 next[1] = 0（1-indexed）
- C++ STL 未规定 `std::string::find` 的具体算法

#### Python 实现

👉 [Python 实现](../python/algorithms/string_algorithm/kmp.py)

```python

def kmp_search(text: str, pattern: str) -> List[int]:
```

### Rabin-Karp 算法

#### 概念说明

Rabin-Karp 算法使用哈希函数对模式串和主串的子串进行快速比较，仅当哈希值匹配时才进行逐字符验证。

#### 权威参考来源

- Richard M. Karp, Michael O. Rabin, "Efficient Randomized Pattern-Matching Algorithms", *IBM Journal of Research and Development*, 1987
- 《算法导论》第 32 章，32.2 节 "The Rabin-Karp algorithm"

#### 学习前置知识

- 哈希函数
- 模运算
- 滚动哈希

#### 实现原理

1. 计算模式串的哈希值和主串第一个窗口的哈希值
2. 滑动窗口时，利用滚动哈希公式 O(1) 更新哈希值
3. 哈希值匹配时进行逐字符验证，避免哈希碰撞导致的误判

#### 核心代码片段

👉 [完整源码](../src/string_algorithm/rabin_karp.cpp)

```cpp
std::vector<int> rabinKarpSearch(const std::string& text, const std::string& pattern) {
    std::vector<int> positions;
    int n = text.length();
    int m = pattern.length();

    if (m > n || m == 0) return positions;

    long long h = 1;
    for (int i = 0; i < m - 1; i++) {
        h = (h * BASE) % MOD;
    }

    long long patternHash = 0;
    long long textHash = 0;
    for (int i = 0; i < m; i++) {
        patternHash = (patternHash * BASE + pattern[i]) % MOD;
        textHash = (textHash * BASE + text[i]) % MOD;
    }

    for (int i = 0; i <= n - m; i++) {
        if (patternHash == textHash) {
            bool match = true;
            for (int j = 0; j < m; j++) {
                if (text[i + j] != pattern[j]) {
                    match = false;
                    break;
                }
            }
            if (match) {
                positions.push_back(i);
            }
        }

        if (i < n - m) {
            textHash = (textHash - text[i] * h % MOD + MOD) % MOD;
            textHash = (textHash * BASE + text[i + m]) % MOD;
        }
    }

    return positions;
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 平均时间复杂度 | O(n+m) |
| 最坏时间复杂度 | O(nm)（大量哈希碰撞时） |
| 空间复杂度 | O(1) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | Rabin-Karp | STL 未提供 Rabin-Karp |
| 多模式匹配 | 支持（每个模式串独立计算哈希） | 不支持 |
| 哈希碰撞 | 需二次验证 | 不适用 |

#### 已知变体和演进

- **双哈希/多哈希**：使用多个不同的 BASE 和 MOD 减少碰撞概率
- **滚动哈希类**：本项目 [rolling_hash.cpp](../src/string_algorithm/rolling_hash.cpp) 封装了独立的滚动哈希数据结构
- **二维 Rabin-Karp**：用于二维模式匹配
- **Karp-Rabin 指纹**：用于文档相似度检测和去重

#### 版本/平台相关注意事项

- BASE 和 MOD 的选择影响碰撞概率：常用 BASE=256/131，MOD=10^9+7/10^9+9
- 单哈希存在碰撞风险，生产环境建议使用双哈希
- 本项目使用 `long long` 存储哈希值，避免 32 位溢出

#### Python 实现

👉 [Python 实现](../python/algorithms/string_algorithm/rabin_karp.py)

```python
def rabin_karp_search(text: str, pattern: str) -> List[int]:
    n, m = len(text), len(pattern)
    if m == 0 or m > n:
        return []
    positions: List[int] = []
    h = 1
    for _ in range(m - 1):
        h = (h * BASE) % MOD
    pattern_hash, text_hash = 0, 0
    for i in range(m):
        pattern_hash = (pattern_hash * BASE + ord(pattern[i])) % MOD
        text_hash = (text_hash * BASE + ord(text[i])) % MOD
    for i in range(n - m + 1):
        if pattern_hash == text_hash:
            if text[i:i + m] == pattern:
                positions.append(i)
        if i < n - m:
            text_hash = (text_hash - ord(text[i]) * h % MOD + MOD) % MOD
            text_hash = (text_hash * BASE + ord(text[i + m])) % MOD
    return positions
```

### Boyer-Moore 算法

#### 概念说明

Boyer-Moore 算法从模式串末尾向前匹配，利用坏字符规则跳过不必要的比较。本项目实现了简化版（仅坏字符规则）。

#### 权威参考来源

- Robert S. Boyer, J Strother Moore, "A Fast String Searching Algorithm", *Communications of the ACM*, Vol.20, No.10, 1977 — Boyer-Moore 算法的原始论文
- 《算法导论》第 32 章，练习题中讨论了坏字符规则

#### 学习前置知识

- 暴力字符串匹配
- 从右向左匹配的思想
- 坏字符规则与好后缀规则

#### 实现原理

1. **构建坏字符表**：记录每个字符在模式串中最后出现的位置
2. **从右向左匹配**：模式串与主串对齐后，从模式串末尾开始比较
3. **坏字符规则**：失配时，根据主串中失配字符在模式串中的位置决定滑动距离

#### 核心代码片段

👉 [完整源码](../src/string_algorithm/boyer_moore.cpp)

```cpp
std::vector<int> buildBadCharTable(const std::string& pattern) {
    int m = pattern.length();
    std::vector<int> badChar(ALPHABET_SIZE, -1);

    for (int i = 0; i < m; i++) {
        badChar[(unsigned char)pattern[i]] = i;
    }

    return badChar;
}

std::vector<int> boyerMooreSearch(const std::string& text, const std::string& pattern) {
    std::vector<int> positions;
    int n = text.length();
    int m = pattern.length();

    if (m == 0 || m > n) return positions;

    std::vector<int> badChar = buildBadCharTable(pattern);

    int s = 0;
    while (s <= n - m) {
        int j = m - 1;

        while (j >= 0 && pattern[j] == text[s + j]) {
            j--;
        }

        if (j < 0) {
            positions.push_back(s);
            int shift = (s + m < n) ? m - badChar[(unsigned char)text[s + m]] : 1;
            s += (shift > 0) ? shift : 1;
        } else {
            int shift = j - badChar[(unsigned char)text[s + j]];
            s += (shift > 0) ? shift : 1;
        }
    }

    return positions;
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 平均时间复杂度 | O(n/m) ~ O(n) |
| 最坏时间复杂度 | O(nm)（仅坏字符规则时） |
| 预处理时间 | O(m + k) |
| 空间复杂度 | O(k) |

其中 k 为字符集大小。完整 Boyer-Moore（含好后缀规则）最坏 O(n)。

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 简化 Boyer-Moore（仅坏字符） | `std::string::find` 实现未规定 |
| 完整性 | 仅坏字符规则 | 不适用 |
| 实际地位 | 教学演示 | 完整 BM 算法是实际工程中应用最广泛的字符串匹配算法之一 |

#### 已知变体和演进

- **Boyer-Moore-Horspool**（1980）：简化版 BM，仅使用坏字符规则中最后一个字符，实现更简单
- **Boyer-Moore-Horspool-Sunday**（1990）：Sunday 算法，利用匹配窗口后一个字符的信息
- **完整 Boyer-Moore**：加入好后缀规则，最坏 O(n)
- **Aho-Corasick**：多模式匹配场景的替代方案

#### 版本/平台相关注意事项

- 本项目仅实现坏字符规则，最坏情况为 O(nm)
- 坏字符表大小为字符集大小（本项目 ALPHABET_SIZE），ASCII 场景为 256
- 完整 BM 算法的好后缀规则实现复杂，实际中 Horspool 简化版更常用
- `std::string::find` 在 GCC libstdc++ 中通常使用简化版 BM 或 KMP

```python
    while s <= n - m:
        j = m - 1
        while j >= 0 and pattern[j] == text[s + j]:
            j -= 1
        if j < 0:
            positions.append(s)
            shift = m - bad_char[ord(text[s + m])] if s + m < n else 1
            s += shift if shift > 0 else 1
        else:
            shift = j - bad_char[ord(text[s + j])]
            s += shift if shift > 0 else 1
    return positions
```

**与 C/C++ 实现差异**：Python 版本使用 ord() 索引坏字符表，无需 unsigned char 强制转换。

### Manacher 算法

#### 概念说明

Manacher 算法用于在线性时间内找到字符串中的最长回文子串。通过在字符间插入特殊字符统一奇偶长度处理，并利用已计算的回文半径加速后续计算。

#### 权威参考来源

- Glenn K. Manacher, "A New Linear-Time 'On-Line' Algorithm for Finding the Smallest Initial Palindrome of a String", *Journal of the ACM*, 1975
- 《算法导论》未收录此算法，但它是回文问题的标准解法

#### 学习前置知识

- 回文的定义
- 中心扩展法求回文
- 对称性利用

#### 实现原理

1. **预处理**：在字符间插入 `#`，统一奇偶长度回文处理
2. **维护右边界**：记录当前最右回文边界 right 及其中心 center
3. **利用对称性**：若当前位置在 right 内，利用镜像位置加速计算
4. **中心扩展**：在加速基础上继续尝试扩展回文半径

#### 核心代码片段

👉 [完整源码](../src/string_algorithm/manacher.cpp)

```cpp
std::vector<int> manacher(const std::string& s) {
    std::string t = "#";
    for (char c : s) {
        t += c;
        t += '#';
    }
    int n = t.length();
    std::vector<int> p(n, 0);

    int center = 0, right = 0;

    for (int i = 0; i < n; i++) {
        if (i < right) {
            int mirror = 2 * center - i;
            p[i] = std::min(right - i, p[mirror]);
        }

        while (i - p[i] - 1 >= 0 && i + p[i] + 1 < n && t[i - p[i] - 1] == t[i + p[i] + 1]) {
            p[i]++;
        }

        if (i + p[i] > right) {
            center = i;
            right = i + p[i];
        }
    }

    return p;
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(n) |
| 空间复杂度 | O(n) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | Manacher | STL 未提供回文查找接口 |
| 功能 | 最长回文子串 | 需自行实现 |

#### 已知变体和演进

- **中心扩展法**：O(n²) 暴力，Manacher 的前身
- **回文树（Eertree/Palindromic Tree）**：Golubev 2013 年提出，可统计所有不同回文子串
- **回文自动机**：与回文树等价，支持在线构造

#### 版本/平台相关注意事项

### 最长公共子序列 (LCS)

#### 概念说明

最长公共子序列（LCS）问题求两个字符串的最长公共子序列的长度和内容。子序列不要求连续，但要求保持相对顺序。

#### 权威参考来源

- 《算法导论》第 15 章 "Dynamic Programming"，15.4 节 "Longest common subsequence"
- LCS 是动态规划的经典问题之一

#### 学习前置知识

- 动态规划基础
- 二维 DP 表的填写
- 回溯构造解

#### 实现原理

1. 定义 `dp[i][j]` 为 s1 前 i 个字符与 s2 前 j 个字符的 LCS 长度
2. 转移方程：若 `s1[i-1] == s2[j-1]`，则 `dp[i][j] = dp[i-1][j-1] + 1`；否则 `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`
3. 回溯构造 LCS 字符串

#### 核心代码片段

👉 [完整源码](../src/string_algorithm/lcs.cpp)

```cpp
LCSResult longestCommonSubsequence(const std::string& s1, const std::string& s2) {
    int m = s1.length();
    int n = s2.length();
    std::vector<std::vector<int>> dp(m + 1, std::vector<int>(n + 1, 0));

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s1[i - 1] == s2[j - 1]) {
                dp[i][j] = dp[i - 1][j - 1] + 1;
            } else {
                dp[i][j] = std::max(dp[i - 1][j], dp[i][j - 1]);
            }
        }
    }

    std::string lcs;
    int i = m, j = n;
    while (i > 0 && j > 0) {
        if (s1[i - 1] == s2[j - 1]) {
            lcs = s1[i - 1] + lcs;
            i--;
            j--;
        } else if (dp[i - 1][j] > dp[i][j - 1]) {
            i--;
        } else {
            j--;
        }
    }

    return {dp[m][n], lcs};
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(mn) |
| 空间复杂度 | O(mn)（可优化至 O(min(m,n))，但无法回溯构造解） |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 经典 DP | STL 未提供 LCS 接口 |
| 空间优化 | O(mn) | 不适用 |

#### 已知变体和演进

- **空间优化 LCS**：仅保留两行，O(min(m,n)) 空间
- **Hirschberg 算法**（1975）：O(min(m,n)) 空间且可构造解，利用分治

### 滚动哈希

#### 概念说明

滚动哈希（Rolling Hash）允许在 O(1) 时间内计算滑动窗口的哈希值，是 Rabin-Karp 算法的核心数据结构。

#### 权威参考来源

- Karp, Rabin (1987)，滚动哈希作为 Rabin-Karp 算法的核心组件
- 多项式滚动哈希是应用最广泛的滚动哈希方案

#### 学习前置知识

- 哈希函数
- 模运算
- 多项式求值

#### 实现原理

1. 使用多项式哈希：`H = s[0]*BASE^(m-1) + s[1]*BASE^(m-2) + ... + s[m-1]`
2. 滑动窗口时：`H' = (H - s[out]*BASE^(m-1)) * BASE + s[in]`
3. 所有运算对 MOD 取模，防止溢出

#### 核心代码片段

👉 [完整源码](../src/string_algorithm/rolling_hash.cpp)

```cpp
class RollingHash {
private:
    long long hash;
    long long power;
    int windowSize;

public:
    RollingHash() : hash(0), power(1), windowSize(0) {}

    long long computeHash(const std::string& s) {
        int n = s.length();
        hash = 0;
        power = 1;
        windowSize = n;

        for (int i = 0; i < n; i++) {
            hash = (hash * BASE + s[i]) % MOD;
            if (i < n - 1) {
                power = (power * BASE) % MOD;
            }
        }
        return hash;
    }

    long long roll(char outChar, char inChar) {
        hash = (hash - outChar * power % MOD + MOD) % MOD;
        hash = (hash * BASE + inChar) % MOD;
        return hash;
    }

    long long getHash() const { return hash; }
    int getWindowSize() const { return windowSize; }
};
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 初始化时间复杂度 | O(m) |
| 滚动更新时间复杂度 | O(1) |
| 空间复杂度 | O(1) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 多项式滚动哈希 | STL 未提供滚动哈希接口 |
| 封装 | 独立类 | 不适用 |

#### 已知变体和演进

- **多项式滚动哈希**：本项目实现，最常用
- **Cyclic Polynomial Hash（Buzhash）**：使用循环移位代替乘法，速度更快
- **Rabin Fingerprint**：不可逆哈希，用于增量哈希计算

#### 版本/平台相关注意事项

- BASE 和 MOD 的选择至关重要：BASE 应大于字符集大小，MOD 应为大素数
- 常用参数：BASE=131/256，MOD=10^9+7/10^9+9
- 减法后加 MOD 防止负数：`(hash - x % MOD + MOD) % MOD`

## 动态规划

### 概述

动态规划（Dynamic Programming, DP）是将复杂问题分解为重叠子问题，通过存储子问题的解避免重复计算的方法。

**权威参考来源**：
- Richard Bellman, *Dynamic Programming*, Princeton University Press, 1957 — DP 理论的奠基之作
- 《算法导论》第 14 章 "Dynamic Programming: Rod Cutting" 与第 15 章，系统阐述了 DP 的设计方法

**DP 解题步骤**：

1. **定义状态**：明确 `dp[i]` 或 `dp[i][j]` 的含义
2. **推导转移方程**：找到状态之间的关系
3. **确定边界条件**：初始化基础状态
4. **选择遍历顺序**：确保计算当前状态时依赖的状态已计算

### 斐波那契数列

#### 概念说明

斐波那契数列是 DP 的经典入门问题：F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2)。项目实现了四种方法：递归、记忆化搜索、DP 表法和矩阵快速幂。

#### 权威参考来源

- 《算法导论》第 4 章，矩阵快速幂求斐波那契数
- Knuth, *TAOCP* Vol.1, 第 1.2.8 节，斐波那契数与黄金分割的关系

#### 学习前置知识

- 递归
- 数组/向量
- 矩阵乘法（矩阵快速幂）

#### 实现原理

- **递归法**：直接按定义递归，存在大量重复计算
- **记忆化搜索**：自顶向下递归 + 备忘录，避免重复计算
- **DP 表法**：自底向上填表，从 F(0) 逐步计算到 F(n)
- **矩阵快速幂**：利用矩阵乘法的结合律，O(log n) 求解

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/dynamic_programming/fibonacci.cpp)

```cpp
long long fib_dp(int n) {
    if (n <= 1) return n;
    vector<long long> dp(n + 1);
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i <= n; i++)
        dp[i] = dp[i - 1] + dp[i - 2];
    return dp[n];
}

long long fib_matrix(int n) {
    if (n == 0) return 0;
    long long F[2][2] = {{1, 1}, {1, 0}};
    power(F, n - 1);
    return F[0][0];
}
```

### 爬楼梯

#### 概念说明

爬楼梯问题：有 n 级台阶，每次可上 1 级或 2 级，求到达顶部的方案数。本质上就是斐波那契数列的变体。

#### 权威参考来源

- LeetCode 70 "Climbing Stairs"，DP 入门经典题目
- 本质为斐波那契数列的偏移（climb(n) = F(n+1)）

#### 学习前置知识

- 斐波那契数列
- DP 状态定义与转移

#### 实现原理

- 状态定义：`dp[i]` 表示到达第 i 级台阶的方案数
- 转移方程：`dp[i] = dp[i-1] + dp[i-2]`
- 扩展：K 步爬楼梯 `dp[i] = Σ dp[i-j]`（j=1..k）、最小花费爬楼梯

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/dynamic_programming/climbing_stairs.cpp)

```cpp
int climb_stairs_basic(int n) {
    if (n <= 2) return n;
    vector<int> dp(n + 1);
    dp[1] = 1;
    dp[2] = 2;
    for (int i = 3; i <= n; i++)
        dp[i] = dp[i - 1] + dp[i - 2];
    return dp[n];
}

int climb_stairs_optimized(int n) {
    if (n <= 2) return n;
    int prev2 = 1, prev1 = 2;
    for (int i = 3; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

### 最大子数组和

#### 概念说明

最大子数组和问题（Kadane 算法）：给定整数数组，找到和最大的连续子数组。项目还实现了允许空子数组和环形数组的变体。

#### 权威参考来源

- Kadane 算法由 Jay Kadane 在 CMU 提出（约 1984 年），参见 Bentley, *Programming Pearls*
- 《算法导论》第 4 章，分治法求解最大子数组

#### 学习前置知识

- 一维 DP
- 前缀和
- 分治法（对比学习）

#### 实现原理

- **Kadane 算法**：遍历数组，维护当前子数组和 `curr_sum` 和最大和 `max_sum`。若 `curr_sum + nums[i] < nums[i]`，则重新开始子数组
- **环形数组**：最大子数组和 = max(普通最大和, 总和 - 最小子数组和)

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/dynamic_programming/max_subarray.cpp)

```cpp
SubarrayResult kadane(const vector<int>& nums) {
    int n = nums.size();
    if (n == 0) return {0, -1, -1};

    int max_sum = nums[0];
    int curr_sum = nums[0];
    int start = 0, end = 0;
    int temp_start = 0;

    for (int i = 1; i < n; i++) {
        if (curr_sum + nums[i] < nums[i]) {
            curr_sum = nums[i];
            temp_start = i;
        } else {
            curr_sum += nums[i];
        }
        if (curr_sum > max_sum) {
            max_sum = curr_sum;
            start = temp_start;
            end = i;
        }
    }
    return {max_sum, start, end};
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(n) |
| 空间复杂度 | O(1) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | Kadane | STL 未提供最大子数组接口 |
| 分治法 | 另见 [max_subarray_dc.cpp](../src/advanced_algorithm/divide_and_conquer/max_subarray_dc.cpp) | 不适用 |

#### 已知变体和演进

- **分治法**：O(n log n)，见 [max_subarray_dc.cpp](../src/advanced_algorithm/divide_and_conquer/max_subarray_dc.cpp)
- **环形数组**：LeetCode 918，利用总和减最小子数组和
- **二维最大子矩阵**：枚举行区间 + 一维 Kadane

#### 版本/平台相关注意事项

- 环形数组中 `total - min_sum` 可能等于 `total`（即选了整个数组），此时等价于非环形情况

### 最长递增子序列 (LIS)

#### 概念说明

最长递增子序列（LIS）问题求给定数组中最长严格递增子序列的长度。项目实现了 O(n²) DP 和 O(n log n) 二分搜索两种方法，以及 LIS 个数统计。

#### 权威参考来源

- 《算法导论》第 14 章，练习题 14.5-2 讨论 LIS
- Schensted, "Longest Increasing and Decreasing Subsequences", *Canadian Journal of Mathematics*, 1961 — LIS 与排列的关系

#### 学习前置知识

- 动态规划
- 二分查找（lower_bound）
- 耐心排序（Patience Sorting）的概念

#### 实现原理

- **O(n²) DP**：`dp[i]` 表示以 `nums[i]` 结尾的 LIS 长度，枚举 j < i 转移
- **O(n log n) 二分**：维护 tails 数组，`tails[k]` 表示长度为 k+1 的 LIS 的最小末尾元素，用二分查找更新

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/dynamic_programming/lis.cpp)

```cpp
int lis_binary(const vector<int>& nums, vector<int>& seq) {
    int n = nums.size();
    if (n == 0) return 0;
    vector<int> tails;
    vector<int> tail_indices;
    vector<int> parent(n, -1);

    for (int i = 0; i < n; i++) {
        auto it = lower_bound(tails.begin(), tails.end(), nums[i]);
        int pos = it - tails.begin();
        if (it == tails.end()) {
            tails.push_back(nums[i]);
            tail_indices.push_back(i);
        } else {
            *it = nums[i];
            tail_indices[pos] = i;
        }
        if (pos > 0)
            parent[i] = tail_indices[pos - 1];
    }

    int len = tails.size();
    seq.clear();
    for (int i = tail_indices[len - 1]; i != -1; i = parent[i])
        seq.push_back(nums[i]);
    reverse(seq.begin(), seq.end());
    return len;
}
```

#### 时间/空间复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| O(n²) DP | O(n²) | O(n) |
| O(n log n) 二分 | O(n log n) | O(n) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 二分查找 | `std::lower_bound` | `std::lower_bound` 是 LIS O(n log n) 解法的核心 |
| 耐心排序 | tails 数组等价于耐心排序的牌堆 | STL 未提供 LIS 接口 |

#### 已知变体和演进

- **耐心排序（Patience Sorting）**：LIS 的 O(n log n) 解法等价于耐心排序
- **Dilworth 定理**：LIS 长度 = 最小反链划分数
- **俄罗斯套娃信封问题**：LeetCode 354，二维 LIS
- **LIS 个数统计**：LeetCode 673，需额外维护 count 数组

#### 版本/平台相关注意事项

- `std::lower_bound` 要求序列有序，本项目在 tails 数组上使用
- 严格递增用 `lower_bound`，非递减用 `upper_bound`
- O(n log n) 方法构造具体序列需要额外维护 parent 数组

#### Python 实现

👉 [Python 实现](../python/algorithms/dynamic_programming/lis.py)

```python
def lis_length(nums: list[int]) -> int:
    if not nums:
        return 0
    tails: list[int] = []
    for num in nums:
        pos = bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
        else:
            tails[pos] = num
    return len(tails)
```

### 0-1 背包问题

#### 概念说明

0-1 背包问题：给定 n 个物品（每个物品只能选一次）和容量为 W 的背包，求最大价值。项目实现了二维 DP 和一维空间优化两种方法。

#### 权威参考来源

- 《算法导论》第 14 章 "Dynamic Programming: Rod Cutting" 与第 15 章，背包问题的 DP 建模
- Kellerer, Pferschy, Pisinger, *Knapsack Problems*, Springer, 2004 — 背包问题的权威专著

#### 学习前置知识

- 二维 DP
- 状态压缩（空间优化）
- 逆序遍历的原理

#### 实现原理

- 状态定义：`dp[i][w]` 表示前 i 个物品、容量为 w 时的最大价值
- 转移方程：`dp[i][w] = max(dp[i-1][w], dp[i-1][w-wt[i-1]] + val[i-1])`
- 一维优化：逆序遍历 w，保证每个物品只选一次

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/dynamic_programming/knapsack_01.cpp)

```cpp
int knapsack_01_2d(int W, const vector<int>& wt, const vector<int>& val, vector<int>& selected) {
    int n = wt.size();
    vector<vector<int>> dp(n + 1, vector<int>(W + 1, 0));

    for (int i = 1; i <= n; i++) {
        for (int w = 0; w <= W; w++) {
            dp[i][w] = dp[i - 1][w];
            if (w >= wt[i - 1])
                dp[i][w] = max(dp[i][w], dp[i - 1][w - wt[i - 1]] + val[i - 1]);
        }
    }

    selected.clear();
    int w = W;
    for (int i = n; i >= 1; i--) {
        if (dp[i][w] != dp[i - 1][w]) {
            selected.push_back(i - 1);
            w -= wt[i - 1];
        }
    }
    reverse(selected.begin(), selected.end());
    return dp[n][W];
}

int knapsack_01_1d(int W, const vector<int>& wt, const vector<int>& val) {
    int n = wt.size();
    vector<int> dp(W + 1, 0);
    for (int i = 0; i < n; i++)
        for (int w = W; w >= wt[i]; w--)
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);
    return dp[W];
}
```

#### 时间/空间复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 二维 DP | O(nW) | O(nW) |
| 一维优化 | O(nW) | O(W) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 经典 DP | STL 未提供背包问题接口 |
| 空间优化 | 一维数组 | 可使用 `std::vector<int>` |

#### 已知变体和演进

- **完全背包**：物品可重复选取，内层正序遍历
- **多重背包**：物品有数量限制，二进制拆分优化
- **分组背包**：物品分组，每组最多选一个
- **混合背包**：0-1、完全、多重混合

#### 版本/平台相关注意事项

- 0-1 背包一维优化必须逆序遍历，否则同一物品可能被重复选取
- 背包问题时间复杂度为 O(nW)，是伪多项式时间，W 很大时效率低
- 回溯构造选中的物品需使用二维 DP 表

#### Python 实现

👉 [Python 实现](../python/algorithms/dynamic_programming/knapsack_01.py)

```python
def knapsack_01(weights: list[int], values: list[int], capacity: int) -> int:
    n = len(weights)
    dp: list[list[int]] = [[0] * (capacity + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        for w in range(capacity + 1):
            dp[i][w] = dp[i - 1][w]
            if w >= weights[i - 1]:
                dp[i][w] = max(dp[i][w], dp[i - 1][w - weights[i - 1]] + values[i - 1])
    return dp[n][capacity]
```

**与 C/C++ 实现差异**：Python 版本参数顺序为 (weights, values, capacity)，使用列表推导初始化二维 DP 表。

### 完全背包问题

#### 概念说明

完全背包问题：每种物品可以选择无限次，求背包能装的最大价值。与 0-1 背包的区别在于内层循环顺序——正序遍历允许重复选取。

#### 权威参考来源

- 《算法导论》第 15 章，完全背包作为 0-1 背包的扩展

#### 学习前置知识

- 0-1 背包问题
- 遍历顺序对 DP 的影响

#### 实现原理

- 转移方程：`dp[i][w] = max(dp[i-1][w], dp[i][w-wt[i-1]] + val[i-1])`
- 一维优化：正序遍历 w，`dp[w] = max(dp[w], dp[w-wt[i]] + val[i])`

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/dynamic_programming/knapsack_complete.cpp)

```cpp
int knapsack_complete_1d(int W, const vector<int>& wt, const vector<int>& val) {
    int n = wt.size();
    vector<int> dp(W + 1, 0);
    for (int i = 0; i < n; i++)
        for (int w = wt[i]; w <= W; w++)
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);
    return dp[W];
}
```

#### 时间/空间复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 二维 DP | O(nW) | O(nW) |
| 一维优化 | O(nW) | O(W) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 与 0-1 背包的区别 | 内层正序 vs 逆序 | STL 未提供背包接口 |

#### 已知变体和演进

- **完全背包 ↔ 0-1 背包**：仅内层遍历顺序不同
- **硬币找零**：完全背包的特例（价值=面值，求最小硬币数）
- **组合数/排列数**：完全背包框架下的计数问题

#### 版本/平台相关注意事项

- 完全背包一维优化必须正序遍历，与 0-1 背包的逆序遍历形成对比
- 0-1 背包的 `dp[i][w]` 依赖 `dp[i-1][w-wt]`（上一行），完全背包依赖 `dp[i][w-wt]`（当前行）

👉 [Python 实现](../python/algorithms/dynamic_programming/knapsack_complete.py)

```python
def knapsack_complete(weights: list[int], values: list[int], capacity: int) -> int:
    n = len(weights)
    dp: list[int] = [0] * (capacity + 1)
    for i in range(n):
        for w in range(weights[i], capacity + 1):
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[capacity]
```

**与 C/C++ 实现差异**：Python 版本仅实现一维空间优化版本，内层正序遍历体现完全背包特性。

### 硬币找零

#### 概念说明

硬币找零问题：给定不同面值的硬币和目标金额，求凑出目标金额的最少硬币数、组合数或排列数。

#### 权威参考来源

- 《算法导论》第 15 章，硬币找零作为 DP 的典型应用
- 完全背包问题的特例

#### 学习前置知识

- 完全背包问题
- DP 初始化技巧（求最小值初始化为 INF，求方案数初始化为 0）

#### 实现原理

- **最少硬币数**：`dp[i] = min(dp[i], dp[i-c] + 1)`，完全背包框架
- **组合数**：外层遍历硬币（保证顺序无关），`dp[i] += dp[i-c]`
- **排列数**：外层遍历金额（允许不同顺序），`dp[i] += dp[i-c]`

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/dynamic_programming/coin_change.cpp)

```cpp
int coin_change_min(const vector<int>& coins, int amount) {
    vector<int> dp(amount + 1, INT_MAX);
    dp[0] = 0;
    for (int i = 1; i <= amount; i++)
        for (int c : coins)
            if (c <= i && dp[i - c] != INT_MAX)
                dp[i] = min(dp[i], dp[i - c] + 1);
    return dp[amount] == INT_MAX ? -1 : dp[amount];
}

int coin_change_combinations(const vector<int>& coins, int amount) {
    vector<int> dp(amount + 1, 0);
    dp[0] = 1;
    for (int c : coins)
        for (int i = c; i <= amount; i++)
            dp[i] += dp[i - c];
    return dp[amount];
}
```

| 排列数 | O(n·amount) | O(amount) |

### 编辑距离

#### 概念说明

编辑距离（Levenshtein Distance）衡量两个字符串的相似度，定义为将一个字符串转换为另一个所需的最少单字符操作（插入、删除、替换）次数。

#### 权威参考来源

- Vladimir I. Levenshtein, "Binary codes capable of correcting deletions, insertions, and reversals", *Soviet Physics Doklady*, 1966
- 《算法导论》第 15 章

#### 学习前置知识

- 二维 DP
- 字符串操作
- 空间优化技巧

#### 实现原理

- 状态定义：`dp[i][j]` 为 s1 前 i 个字符与 s2 前 j 个字符的编辑距离
- 转移方程：若 `s1[i-1] == s2[j-1]`，`dp[i][j] = dp[i-1][j-1]`；否则 `dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])`
- 空间优化：仅保留两行，O(min(m,n))

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/dynamic_programming/edit_distance.cpp)

```cpp
int edit_distance_2d(const string& s1, const string& s2) {
    int m = s1.size(), n = s2.size();
    vector<vector<int>> dp(m + 1, vector<int>(n + 1));

    for (int i = 0; i <= m; i++) dp[i][0] = i;
    for (int j = 0; j <= n; j++) dp[0][j] = j;

    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++) {
            if (s1[i - 1] == s2[j - 1])
                dp[i][j] = dp[i - 1][j - 1];
            else
                dp[i][j] = 1 + min({dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1]});
        }
    return dp[m][n];
}
```

#### 时间/空间复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 二维 DP | O(mn) | O(mn) |
| 空间优化 | O(mn) | O(min(m,n)) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 经典 DP | STL 未提供编辑距离接口 |
| 操作追踪 | 支持 | 不适用 |

#### 已知变体和演进

- **Damerau-Levenshtein 距离**：增加相邻字符交换操作
- **加权编辑距离**：不同操作赋予不同权重
- **最长公共子序列距离**：删除+插入，不允许替换
- **BK 树**：利用编辑距离的三角不等式加速近似匹配

#### 版本/平台相关注意事项

- 空间优化版本无法回溯构造操作序列
- `min({a, b, c})` 需要 C++11 的 `std::initializer_list` 重载

#### Python 实现

👉 [Python 实现](../python/algorithms/dynamic_programming/edit_distance.py)

```python
def edit_distance(word1: str, word2: str) -> int:
    m, n = len(word1), len(word2)
    dp: list[list[int]] = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i - 1] == word2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = 1 + min(dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1])
    return dp[m][n]
```

**与 C/C++ 实现差异**：Python 版本使用 min() 直接接受多参数，无需 initializer_list 重载。

### 打家劫舍

#### 概念说明

打家劫舍问题：给定一排房屋（每间有金额），不能抢劫相邻房屋，求最大金额。项目实现了线性排列和环形排列两种变体。

#### 权威参考来源

- LeetCode 198/213/337，DP 经典题目系列
- 本质为带约束的子集选取问题

#### 学习前置知识

- 一维 DP
- 状态转移与约束条件

#### 实现原理

- 状态定义：`dp[i]` 表示前 i 间房屋能获得的最大金额
- 转移方程：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`
- 环形排列：首尾不能同时选，拆分为 [0, n-2] 和 [1, n-1] 两个子问题

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/dynamic_programming/house_robber.cpp)

```cpp
int rob_basic_optimized(const vector<int>& nums) {
    int n = nums.size();
    if (n == 0) return 0;
    if (n == 1) return nums[0];
    int prev2 = nums[0];
    int prev1 = max(nums[0], nums[1]);
    for (int i = 2; i < n; i++) {
        int curr = max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

int rob_circular(const vector<int>& nums) {
    int n = nums.size();
    if (n == 0) return 0;
    if (n == 1) return nums[0];
    if (n == 2) return max(nums[0], nums[1]);

    auto rob_range = [](const vector<int>& arr, int l, int r) -> int {
        int prev2 = arr[l];
        int prev1 = max(arr[l], arr[l + 1]);
        for (int i = l + 2; i <= r; i++) {
            int curr = max(prev1, prev2 + arr[i]);
            prev2 = prev1;
            prev1 = curr;
        }
        return prev1;
    };

    return max(rob_range(nums, 0, n - 2), rob_range(nums, 1, n - 1));
}
```

| 空间优化 | O(n) | O(1) |
| 环形排列 | O(n) | O(1) |

```python
def rob(nums: list[int]) -> int:
    n = len(nums)
    if n == 0:
        return 0
    if n == 1:
        return nums[0]
    prev2, prev1 = nums[0], max(nums[0], nums[1])
    for i in range(2, n):
        curr = max(prev1, prev2 + nums[i])
        prev2, prev1 = prev1, curr
    return prev1
```

### 活动选择

#### 概念说明

```cpp
vector<Activity> activity_selection(vector<Activity>& acts) {
```

### 分数背包

#### 概念说明

分数背包问题：物品可以分割，求背包能装的最大价值。贪心策略为按单位价值（价值/重量）降序选取。

#### 权威参考来源

- 《算法导论》第 16 章，16.2 节，证明了分数背包的贪心选择性质
- 0-1 背包不具有贪心选择性质

#### 学习前置知识

- 贪心算法
- 排序
- 0-1 背包问题（对比学习）

#### 实现原理

1. 计算每个物品的单位价值 ratio = value / weight
2. 按单位价值降序排序
3. 依次选取物品，若能完整放入则全部取入，否则取部分

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/greedy/fractional_knapsack.cpp)

```cpp
double fractional_knapsack(int W, vector<Item>& items) {
    sort(items.begin(), items.end(), [](const Item& a, const Item& b) {
        return a.ratio > b.ratio;
    });

    double total_value = 0.0;
    double remaining = W;

    for (auto& item : items) {
        if (remaining <= 0) break;
        double take = min(item.weight, remaining);
        double take_value = take * item.ratio;
        remaining -= take;
        total_value += take_value;
    }
    return total_value;
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(n log n)（排序） |
| 空间复杂度 | O(1) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 贪心 | STL 未提供背包接口 |
| 与 0-1 背包对比 | 分数背包贪心最优 | 0-1 背包贪心不一定最优 |

#### 已知变体和演进

- **0-1 背包贪心（非最优）**：本项目提供了对比实现，证明贪心对 0-1 背包不适用

#### Python 实现

👉 [Python 实现](../python/algorithms/greedy/fractional_knapsack.py)

```python
def fractional_knapsack(weights: list[float], values: list[float], capacity: float) -> float:
    items = sorted(
        zip(weights, values),
        key=lambda x: x[1] / x[0],
        reverse=True,
    )
    total_value = 0.0
    remaining = capacity
    for w, v in items:
        if remaining <= 0:
            break
        take = min(w, remaining)
        total_value += take * (v / w)
        remaining -= take
    return total_value
```

**与 C/C++ 实现差异**：Python 版本使用 zip+sorted 替代结构体排序，lambda 表达式计算单位价值。

### 区间调度

#### 概念说明

```cpp
int min_meeting_rooms(vector<pair<int, int>>& intervals) {
    vector<Event> events;
    for (int i = 0; i < (int)intervals.size(); i++) {
        events.push_back({intervals[i].first, 1, i});
        events.push_back({intervals[i].second, -1, i});
    }

    sort(events.begin(), events.end(), [](const Event& a, const Event& b) {
        if (a.time != b.time) return a.time < b.time;
        return a.type < b.type;
    });

    int rooms = 0, max_rooms = 0;
    for (auto& e : events) {
        rooms += e.type;
        max_rooms = max(max_rooms, rooms);
    }
    return max_rooms;
}
```

### 全排列

#### 概念说明

全排列问题：给定一组元素，生成所有可能的排列。项目实现了交换法和 used 数组法两种方式。

#### 权威参考来源

- 《算法导论》，排列生成作为回溯法的入门
- Heap's Algorithm（1963）：另一种高效的排列生成算法

#### 学习前置知识

- 递归
- 回溯法的基本框架
- 去重策略

#### 实现原理

- **交换法**：从位置 start 开始，依次与后面每个位置交换，递归处理 start+1
- **used 数组法**：维护当前排列和已使用标记，枚举未使用元素，支持去重

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/backtracking/permutations.cpp)

```cpp
class PermutationsUsed {
    vector<vector<int>> results;
    vector<int> current;
    vector<bool> used;

    void backtrack(const vector<int>& nums) {
        if ((int)current.size() == (int)nums.size()) {
            results.push_back(current);
            return;
        }
        for (int i = 0; i < (int)nums.size(); i++) {
            if (used[i]) continue;
            if (i > 0 && nums[i] == nums[i - 1] && !used[i - 1]) continue;
            used[i] = true;
            current.push_back(nums[i]);
            backtrack(nums);
            current.pop_back();
            used[i] = false;
        }
    }

public:
    vector<vector<int>> solve(vector<int> nums) {
        results.clear();
        current.clear();
        used.assign(nums.size(), false);
        sort(nums.begin(), nums.end());
        backtrack(nums);
        return results;
    }
};
```

#### 时间/空间复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 交换法 | O(n · n!) | O(n) |
| used 数组法 | O(n · n!) | O(n) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 回溯 | `std::next_permutation` 生成下一个排列 |
| 去重 | used 数组 + 排序 | `std::next_permutation` 天然去重（需先排序） |
| 生成方式 | 一次性生成全部 | 逐个生成，更节省内存 |

#### 已知变体和演进

- **Heap's Algorithm**：最少交换次数的排列生成算法
- **字典序排列**：`std::next_permutation` 使用此方法
- **带重复元素的排列**：used 数组法支持去重

#### 版本/平台相关注意事项

- `std::next_permutation` 要求序列已排序，且按字典序生成
- 交换法不保证字典序
- used 数组去重条件 `i > 0 && nums[i] == nums[i-1] && !used[i-1]` 是关键

#### Python 实现

👉 [Python 实现](../python/algorithms/backtracking/permutations.py)

```python
def permute(nums: list[int]) -> list[list[int]]:
    results: list[list[int]] = []
    def backtrack(start: int) -> None:
        if start == len(nums):
            results.append(nums[:])
            return
        for i in range(start, len(nums)):
            nums[start], nums[i] = nums[i], nums[start]
            backtrack(start + 1)
            nums[start], nums[i] = nums[i], nums[start]
    backtrack(0)
    return results
```

**与 C/C++ 实现差异**：Python 版本使用交换法+闭包实现，无需类封装和 used 数组。

### 组合

#### 概念说明

组合问题：从 n 个元素中选 k 个，生成所有可能的组合。项目实现了基本组合、可重复组合和数组组合（含去重）。

#### 权威参考来源

- 《算法导论》，组合生成作为回溯法的应用
- 组合数 C(n,k) = n! / (k! · (n-k)!)

#### 学习前置知识

- 回溯法
- 组合数学基础

#### 实现原理

- **基本组合**：从 start 开始枚举，递归选择下一个元素
- **可重复组合**：递归时 start 不递增（允许重复选取同一元素）
- **数组组合**：先排序，枚举时跳过重复元素

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/backtracking/combinations.cpp)

```cpp
class Combinations {
    vector<vector<int>> results;
    vector<int> current;

    void backtrack(int n, int k, int start) {
        if ((int)current.size() == k) {
            results.push_back(current);
            return;
        }
        for (int i = start; i <= n - (k - (int)current.size()) + 1; i++) {
            current.push_back(i);
            backtrack(n, k, i + 1);
            current.pop_back();
        }
    }

public:
    vector<vector<int>> solve(int n, int k) {
        results.clear();
        current.clear();
        backtrack(n, k, 1);
        return results;
    }
};
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(C(n,k) · k) |
| 空间复杂度 | O(k)（递归深度） |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 回溯 | 无直接对应 |

#### 已知变体和演进

- **可重复组合**：C(n+k-1, k)
- **数组组合（含去重）**：排序 + 跳过相同元素
- **剪枝优化**：`i <= n - (k - current.size()) + 1` 提前终止不可能的分支

#### 版本/平台相关注意事项

- 剪枝条件 `n - (k - current.size()) + 1` 确保剩余元素足够选满 k 个
- 去重条件 `i > start && arr[i] == arr[i-1]` 中的 `i > start` 不可省略

#### Python 实现

👉 [Python 实现](../python/algorithms/backtracking/combinations.py)

```python
def combine(n: int, k: int) -> list[list[int]]:
    results: list[list[int]] = []
    current: list[int] = []
    def backtrack(start: int) -> None:
        if len(current) == k:
            results.append(current[:])
            return
        for i in range(start, n - (k - len(current)) + 2):
            current.append(i)
            backtrack(i + 1)
            current.pop()
    backtrack(1)
    return results
```

**与 C/C++ 实现差异**：Python 版本使用闭包替代类封装，列表切片 current[:] 复制当前组合。

### 子集和

#### 概念说明

子集和问题：给定一组正整数和目标和，找出所有和等于目标的子集。项目实现了带剪枝去重、无剪枝和仅计数三种方式。

#### 实现原理

1. 排序数组（剪枝前提）
2. 递归枚举每个元素选或不选
3. 剪枝：当前和 + 当前元素 > 目标时跳过（数组已排序）
4. 去重：跳过与前一元素相同的元素

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/backtracking/subset_sum.cpp)

```cpp
class SubsetSum {
    vector<vector<int>> results;
    vector<int> current;

    void backtrack(const vector<int>& nums, int target, int start, int curr_sum) {
        if (curr_sum == target) {
            results.push_back(current);
            return;
        }
        for (int i = start; i < (int)nums.size(); i++) {
            if (curr_sum + nums[i] > target) break;
            if (i > start && nums[i] == nums[i - 1]) continue;
            current.push_back(nums[i]);
            backtrack(nums, target, i + 1, curr_sum + nums[i]);
            current.pop_back();
        }
    }

public:
    vector<vector<int>> solve(vector<int> nums, int target) {
        results.clear();
        current.clear();
        sort(nums.begin(), nums.end());
        backtrack(nums, target, 0, 0);
        return results;
    }
};
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(2^n)（最坏，无剪枝） |
| 空间复杂度 | O(n)（递归深度） |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 回溯 + 剪枝 | STL 未提供子集和接口 |
| 伪多项式 DP | 未实现 | 可用 DP 求解，O(n·target) |

#### 已知变体和演进

- **伪多项式 DP**：O(n·target)，适用于 target 不大的场景
- **近似算法**：对大规模 NP 完全问题，使用 FPTAS（完全多项式时间近似方案）
- **仅计数版本**：本项目 `SubsetSumCount` 类

- 排序是剪枝的前提，`break` 依赖数组有序
- 去重条件 `i > start && nums[i] == nums[i-1]` 确保同一层不选相同元素

#### Python 实现

👉 [Python 实现](../python/algorithms/backtracking/subset_sum.py)

```python
def subset_sum(nums: list[int], target: int) -> list[list[int]]:
    results: list[list[int]] = []
    current: list[int] = []
    sorted_nums = sorted(nums)
    def backtrack(start: int, curr_sum: int) -> None:
        if curr_sum == target:
            results.append(current[:])
            return
        for i in range(start, len(sorted_nums)):
            if curr_sum + sorted_nums[i] > target:
                break
            if i > start and sorted_nums[i] == sorted_nums[i - 1]:
                continue
            current.append(sorted_nums[i])
            backtrack(i + 1, curr_sum + sorted_nums[i])
            current.pop()
    backtrack(0, 0)
    return results
```

**与 C/C++ 实现差异**：Python 版本使用闭包+内置 sorted 替代类封装，逻辑与 C++ 版本一致。

### N 皇后

#### 概念说明

N 皇后问题：在 n×n 棋盘上放置 n 个皇后，使其互不攻击（同行、同列、同对角线）。项目实现了基本回溯和位运算加速两种方式。

#### 权威参考来源

- 《算法导论》，N 皇后作为回溯法的经典应用
- 位运算加速方法利用了列、对角线的位掩码表示

#### 学习前置知识

- 回溯法
- 位运算
- 对角线冲突的判定

#### 实现原理

- **基本回溯**：逐行放置皇后，检查列和对角线冲突
- **位运算加速**：用三个整数表示列、主对角线、副对角线的占用情况，用位运算快速找到可用位置

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/backtracking/n_queens.cpp)

```cpp
class NQueensBitmask {
    int n;
    int count;

    void backtrack(int row, int cols, int diag1, int diag2) {
        if (row == n) {
            count++;
            return;
        }
        int available = ((1 << n) - 1) & ~(cols | diag1 | diag2);
        while (available) {
            int pos = available & (-available);
            available -= pos;
            backtrack(row + 1, cols | pos, (diag1 | pos) << 1, (diag2 | pos) >> 1);
        }
    }
};
```

#### 时间/空间复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 基本回溯 | O(n!) | O(n) |
| 位运算加速 | O(n!) | O(n) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 回溯 + 位运算 | STL 未提供 N 皇后接口 |
| 位运算加速 | 支持 | 不适用 |

#### 已知变体和演进

- **位运算加速**：本项目 `NQueensBitmask` 类，利用位掩码 O(1) 判断可用位置
- **对称性剪枝**：利用棋盘对称性，只搜索一半解
- **迭代加深**：结合 IDA* 搜索

- 《算法导论》第 4 章 "Divide and Conquer"
- 分治法的三个步骤：分解（Divide）、求解（Conquer）、合并（Combine）

### 最大子数组（分治法）

#### 概念说明

使用分治法求解最大子数组和问题：将数组分为左右两半，最大子数组要么在左半、要么在右半、要么跨越中点。

#### 权威参考来源

- 《算法导论》第 4 章，4.1 节 "Maximum subarray problem"，分治法作为经典范例
- 与 Kadane 算法（O(n)）对比学习

#### 学习前置知识

- 分治法
- 递归
- Kadane 算法（对比学习）

#### 实现原理

1. 将数组分为 [low, mid] 和 [mid+1, high]
2. 递归求左半最大子数组和右半最大子数组
3. 求跨越中点的最大子数组（从 mid 向左扩展 + 从 mid+1 向右扩展）
4. 取三者最大值

#### 核心代码片段

```cpp
MaxSubarrayResult maxSubarrayDC(const vector<int>& nums, int low, int high) {
    if (low == high) {
        return {low, high, nums[low]};
    }

    int mid = (low + high) / 2;
    auto left = maxSubarrayDC(nums, low, mid);
    auto right = maxSubarrayDC(nums, mid + 1, high);
    auto cross = maxCrossingSubarray(nums, low, mid, high);

    if (left.sum >= right.sum && left.sum >= cross.sum) return left;
    if (right.sum >= left.sum && right.sum >= cross.sum) return right;
    return cross;
}
```

|------|-----|
| 时间复杂度 | O(n log n) |
| 空间复杂度 | O(log n)（递归栈） |

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 分治法 | STL 未提供最大子数组接口 |
| 与 Kadane 对比 | O(n log n) | Kadane O(n)，实际更优 |

#### 已知变体和演进

- **Kadane 算法**：O(n)，见 [max_subarray.cpp](../src/advanced_algorithm/dynamic_programming/max_subarray.cpp)
- **分治法**：教学价值高，展示分治思想

#### 版本/平台相关注意事项

- 分治法时间复杂度 O(n log n) 不如 Kadane 的 O(n)，但教学价值高

```python
def max_subarray_dc(nums: list[int]) -> int:
    def cross_sum(left: int, mid: int, right: int) -> int:
        left_sum = float("-inf")
        s = 0
        for i in range(mid, left - 1, -1):
            s += nums[i]
            if s > left_sum:
                left_sum = s
        ...
        return left_sum + right_sum
    def dc(left: int, right: int) -> int:
        if left == right:
            return nums[left]
        mid = left + (right - left) // 2
        return max(dc(left, mid), dc(mid + 1, right), cross_sum(left, mid, right))
    return dc(0, len(nums) - 1)
```

### 最近点对

#### 概念说明

最近点对问题：给定平面上 n 个点，找出距离最近的点对。使用分治法可在 O(n log n) 时间内解决。

#### 权威参考来源

- 《算法导论》第 33 章，33.4 节 "Closest pair of points"
- Shamos, Hoey, "Closest-Point Problems", *FOCS*, 1975 — 最早提出 O(n log n) 算法

#### 学习前置知识

- 分治法
- 递归
- 几何距离计算
- 归并排序（按 y 坐标归并）

#### 实现原理

1. 按 x 坐标排序，将点集分为左右两半
2. 递归求左半和右半的最近点对距离 d
3. 检查跨越中线的点对：仅考虑中线两侧距离 < d 的点，按 y 坐标排序后只需检查每个点后续 7 个点

#### 核心代码片段

👉 [完整源码](../src/advanced_algorithm/divide_and_conquer/closest_pair.cpp)

```cpp
double closestPairDC(vector<Point>& points, int left, int right) {
    if (right - left <= 3) {
        return bruteForce(points, left, right);
    }

    int mid = (left + right) / 2;
    double midX = points[mid].x;

    double dl = closestPairDC(points, left, mid);
    double dr = closestPairDC(points, mid + 1, right);
    double d = min(dl, dr);

    vector<Point> strip;
    for (int i = left; i <= right; i++) {
        if (abs(points[i].x - midX) < d) {
            strip.push_back(points[i]);
        }
    }

    sort(strip.begin(), strip.end(), [](const Point& a, const Point& b) {
        return a.y < b.y;
    });

    for (int i = 0; i < (int)strip.size(); i++) {
        for (int j = i + 1; j < (int)strip.size() && strip[j].y - strip[i].y < d; j++) {
            double dist = distance(strip[i], strip[j]);
            if (dist < d) d = dist;
        }
    }

    return d;
}
```

#### 时间/空间复杂度分析

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(n log n) |
| 空间复杂度 | O(n) |

#### 与 C++ STL 标准库实现的对比

| 维度 | 本项目实现 | C++ STL |
|------|-----------|---------|
| 算法 | 分治法 | STL 未提供最近点对接口 |
| 排序 | `std::sort` | `std::sort` |

#### 已知变体和演进

- **暴力法**：O(n²)，本项目提供了对比实现
- **带归并排序优化**：按 y 坐标归并而非排序，严格 O(n log n)

### 数学算法

#### 概念说明

项目实现了常见的数学算法，包括最大公约数、素数判定、快速幂、矩阵快速幂等。

#### 权威参考来源

- 《算法导论》第 31 章 "Number-Theoretic Algorithms"
- Knuth, *TAOCP* Vol.2, 半数值算法

#### 学习前置知识

- 数论基础
- 模运算
- 矩阵运算

#### 实现原理

- **欧几里得算法（GCD）**：`gcd(a, b) = gcd(b, a % b)`，递归至 `b == 0`；扩展欧几里得可求解 `ax + by = gcd(a, b)` 的整数解
- **素数判定**：试除法 `O(√n)`，6k±1 优化；Miller-Rabin 概率性素性测试，基于 Fermat 小定理的逆否命题
- **埃拉托斯特尼筛法**：从 2 开始标记合数，每个素数 p 标记 p², p²+p, p²+2p, …，时间 O(n log log n)
- **快速幂（模幂）**：二进制拆分指数，`base^exp = ∏ base^(2^i)`（exp 的第 i 位为 1 时累乘），O(log exp)
- **矩阵快速幂**：将递推关系表示为矩阵乘法，再用快速幂加速，如 Fibonacci 的 `[[1,1],[1,0]]^n`
- **模逆元**：扩展欧几里得求 `a^(-1) mod m`，要求 `gcd(a, m) == 1`
- **卡特兰数**：递推 `C(n) = Σ C(i)×C(n-1-i)`，动态规划求解
- **组合数**：递推 `C(n,k) = C(n-1,k-1) + C(n-1,k)`，滚动数组优化空间

👉 [完整源码](../src/math/math_algorithms.cpp)

```cpp
    while (b != 0) {

        b = a % b;
        a = t;

bool millerRabin(int64_t n, int k) {
    if (n < 2) return false;
    if (n < 4) return true;
    if (n % 2 == 0) return false;
    int64_t d = n - 1;
    while (d % 2 == 0) d >>= 1;
    std::mt19937_64 rng(42);
    for (int i = 0; i < k; i++) {
        int64_t a = 2 + rng() % (n - 3);
        if (!millerRabinTest(n, d, a)) return false;
    }
    return true;
}

int64_t fastPow(int64_t base, int64_t exp, int64_t mod) {
    return modPow(base, exp, mod);
}

int64_t fibonacciMatrix(int n) {
    if (n <= 0) return 0;
    if (n == 1) return 1;
    int64_t result[2][2] = {{1, 0}, {0, 1}};
    int64_t base[2][2] = {{1, 1}, {1, 0}};
    int64_t mod = 1000000007;
    int p = n - 1;
    while (p > 0) {
        if (p & 1) matrixMultiply(result, base, mod);
        matrixMultiply(base, base, mod);
        p >>= 1;
    }
    return result[0][0];
}
```

|------|-----------|---------|
| GCD | 手动实现欧几里得 | C++17 `std::gcd`（`<numeric>`） |
| LCM | 手动实现 | C++17 `std::lcm`（`<numeric>`） |
| 快速幂 | 手动实现 | 无标准库支持，需自行实现或使用第三方库 |
| 素数判定 | Miller-Rabin | 无标准库支持 |
| 组合数 | 滚动数组 DP | 无标准库支持 |

#### 已知变体和演进

- **GCD**：欧几里得算法 → 二进制 GCD（Stein 算法），避免取模运算，仅用减法和移位
- **素数判定**：试除法 → Miller-Rabin → AKS 确定性多项式算法（2002）
- **筛法**：埃氏筛 → 欧拉线性筛（每个合数只被最小素因子筛一次）→ 分段筛（大范围）
- **快速幂**：二进制拆分 → 蒙哥马利约减（模数固定时更优）
- **组合数**：递推 → Lucas 定理（大数取模）→ 多项式算法（FFT 加速）

#### 版本/平台相关注意事项

- C++17 `<numeric>` 提供了 `std::gcd` 和 `std::lcm`，推荐优先使用
- `int64_t` 乘法溢出：模乘需用 `__int128`（GCC/Clang 扩展）或快速乘法，MSVC 无 `__int128` 需另寻方案
- Miller-Rabin 的随机数生成器：本项目使用固定种子 `std::mt19937_64 rng(42)`，生产环境应使用 `std::random_device`
- 矩阵快速幂的模数 `1000000007` 是竞赛常用值，实际应用需根据需求调整
