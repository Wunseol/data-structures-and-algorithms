# 07 排序算法模块需求文档

## 1. 模块概述

排序算法是数据结构与算法学习的核心模块之一，也是面试高频考点。本模块旨在系统性地覆盖从基础 O(n²) 排序到高级 O(nlogn) 排序，再到特殊场景 O(n) 排序的完整知识体系，并结合 C++ STL 和 Python 内置排序的实现原理进行深入理解。

**模块目标：**
- 掌握 8 种经典排序算法的原理、实现与性能特征
- 修复现有代码中的已知 Bug
- 补充非比较排序算法（计数排序、基数排序、桶排序）
- 了解工业级排序算法（Timsort、Introsort）
- 建立完整的复杂度分析与稳定性分析体系
- 理解外部排序的基本原理

**实现语言：** C/C++（主要实现语言），Python（辅助验证与对比）

### 2.4 已知 Bug 详细分析

#### Bug 1：Merge 函数边界条件错误（严重）

**位置：** `1_3_Sequential_list_排序.cpp` 第 192-202 行

```cpp
void Merge(RedType R[],RedType T[],int low,int mid,int high)
{
    int i=low,j=mid+1,k=low;
    while(i<=mid&&j<high)       // Bug: j<high 应为 j<=high
    {
        if(R[i].key<=R[j].key) T[k++]=R[i++];
        else T[k++]=R[j++];
    }
    while(i<mid) T[k++]=R[i++]; // Bug: i<mid 应为 i<=mid
    while(j<high) T[k++]=R[j++];// Bug: j<high 应为 j<=high
}
```

**问题分析：**
- `while(i<=mid&&j<high)`：第二个子数组的上界是 `high`，循环条件应为 `j<=high`，当前条件导致 `R[high]` 永远不会被比较
- `while(i<mid)`：第一个子数组的上界是 `mid`，应为 `i<=mid`，当前条件导致 `R[mid]` 可能被遗漏
- `while(j<high)`：同理，应为 `j<=high`

**影响：** 归并排序结果不正确，最后一个元素可能丢失或未被正确归位。

**修复方案：**
```cpp
while(i<=mid&&j<=high)
{
    if(R[i].key<=R[j].key) T[k++]=R[i++];
    else T[k++]=R[j++];
}
while(i<=mid) T[k++]=R[i++];
while(j<=high) T[k++]=R[j++];
```

## 3. 缺失内容

### 3.1 缺失的排序算法

#### 3.1.1 计数排序（Counting Sort）

- **类别：** 非比较排序
- **时间复杂度：** O(n+k)，k 为数据范围
- **空间复杂度：** O(n+k)
- **稳定性：** 稳定
- **适用场景：** 数据范围不大（k << n）的整数排序
- **实现要求：**
  - 基本计数排序实现
  - 稳定版本（使用前缀和）
  - 分析 k 与 n 的关系对性能的影响
- **学习重点：** 理解非比较排序突破 O(nlogn) 下界的原理

#### 3.1.2 基数排序（Radix Sort）

- **类别：** 非比较排序
- **时间复杂度：** O(d×(n+k))，d 为位数，k 为基数
- **空间复杂度：** O(n+k)
- **稳定性：** 稳定（必须使用稳定排序作为子过程）
- **适用场景：** 定长关键字排序（如整数、字符串）
- **实现要求：**
  - LSD（最低位优先）基数排序
  - MSD（最高位优先）基数排序（选做）
  - 使用计数排序作为子过程
- **学习重点：** 理解"按位排序"的思想，以及稳定性对基数排序的重要性

#### 3.1.3 桶排序（Bucket Sort）

- **类别：** 非比较排序
- **时间复杂度：** 平均 O(n+k)，最坏 O(n²)
- **空间复杂度：** O(n+k)
- **稳定性：** 取决于桶内排序算法
- **适用场景：** 数据均匀分布的情况
- **实现要求：**
  - 基于链表的桶排序实现
  - 分析数据分布对性能的影响
- **学习重点：** 理解均匀分布假设的重要性

#### 3.1.4 Timsort（Python/Java 内置排序）

- **类别：** 混合排序算法（归并排序 + 插入排序）
- **时间复杂度：** 最好 O(n)，最坏 O(nlogn)
- **空间复杂度：** O(n)
- **稳定性：** 稳定
- **实现要求：**
  - 不要求完整实现，但需理解核心思想
  - Python 中 `sorted()` 和 `list.sort()` 均使用 Timsort
  - Java 中 `Arrays.sort()` 对对象使用 Timsort（Java 7+）
  - 重点理解 "run" 的概念和 galloping mode
- **学习重点：** 理解工业级排序算法如何利用数据的局部有序性

#### 3.1.5 外部排序（External Sort）

- **类别：** 大规模数据排序
- **适用场景：** 数据量超过内存容量
- **实现要求：**
  - 理解外部排序的基本流程：内部排序 → 置换选择 → 多路归并
  - 败者树（Loser Tree）的实现
  - 最佳归并树的概念
  - 不要求完整实现，但需能描述流程并画出示意图
- **学习重点：** 理解磁盘 I/O 对排序算法设计的影响

### 3.2 缺失的分析内容

#### 3.2.1 排序算法稳定性分析

对每种排序算法进行稳定性分析，需说明：
- 是否稳定
- 稳定/不稳定的根本原因（通过具体反例说明）
- 稳定性在实际应用中的意义（如多关键字排序）

| 算法 | 稳定性 | 原因分析 |
|------|--------|---------|
| 直接插入排序 | 稳定 | 从后向前比较，相等时不移动 |
| 折半插入排序 | 稳定 | 折半查找定位后统一后移，相等元素保持原序 |
| 希尔排序 | **不稳定** | 不同增量的分组可能打破相对顺序 |
| 冒泡排序 | 稳定 | 相邻交换，相等时不交换 |
| 快速排序 | **不稳定** | 划分过程中跨越式交换可能改变相对顺序 |
| 简单选择排序 | **不稳定** | 交换可能跨越相等元素 |
| 堆排序 | **不稳定** | 筛选过程可能改变相等元素的相对顺序 |
| 归并排序 | 稳定 | 归并时左半部分优先 |
| 计数排序 | 稳定 | 使用前缀和的版本可保证稳定性 |
| 基数排序 | 稳定 | 要求子排序必须稳定 |
| 桶排序 | 取决于桶内排序 | 若桶内使用插入排序则稳定 |

#### 3.2.2 时间/空间复杂度对比表

| 算法 | 最好时间 | 平均时间 | 最坏时间 | 空间复杂度 | 稳定性 |
|------|---------|---------|---------|-----------|--------|
| 直接插入排序 | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| 折半插入排序 | O(nlogn) | O(n²) | O(n²) | O(1) | 稳定 |
| 希尔排序 | O(nlogn) | 取决于增量序列 | 取决于增量序列 | O(1) | 不稳定 |
| 冒泡排序 | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| 快速排序 | O(nlogn) | O(nlogn) | O(n²) | O(logn) | 不稳定 |
| 简单选择排序 | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 |
| 堆排序 | O(nlogn) | O(nlogn) | O(nlogn) | O(1) | 不稳定 |
| 归并排序 | O(nlogn) | O(nlogn) | O(nlogn) | O(n) | 稳定 |
| 计数排序 | O(n+k) | O(n+k) | O(n+k) | O(n+k) | 稳定 |
| 基数排序 | O(d(n+k)) | O(d(n+k)) | O(d(n+k)) | O(n+k) | 稳定 |
| 桶排序 | O(n+k) | O(n+k) | O(n²) | O(n+k) | 取决于 |

> **注意：** 折半插入排序的最好时间 O(nlogn) 仅指比较次数，移动次数仍为 O(n²)，因此总体时间复杂度仍为 O(n²)。

#### 3.2.3 最好/最坏/平均情况分析

需对每种排序算法详细说明：
- **最好情况：** 什么输入导致？时间复杂度？
- **最坏情况：** 什么输入导致？时间复杂度？
- **平均情况：** 假设输入随机分布时的期望时间复杂度
- **典型反例：** 导致最坏情况的具体输入模式

**重点分析：**
- 快速排序的最坏情况（已排序输入 + 首元素作枢轴）及优化策略（三数取中、随机化）
- 堆排序为何最坏情况仍为 O(nlogn)
- 归并排序为何各情况一致

## 6. 学习路径

### 第一阶段：O(n²) 基础排序（1-2 周）

```
直接插入排序 → 折半插入排序 → 冒泡排序 → 简单选择排序
```

**学习重点：**
- 理解"插入"、"交换"、"选择"三种基本排序思想
- 掌握最好/最坏/平均情况分析
- 理解稳定性概念
- 练习：手动模拟每种排序的执行过程

### 第二阶段：O(nlogn) 高级排序（2-3 周）

```
希尔排序 → 快速排序 → 堆排序 → 归并排序
```

**学习重点：**
- 希尔排序：理解增量序列对性能的影响
- 快速排序：理解分治思想，掌握 Partition 的多种实现
- 堆排序：理解堆的性质与筛选过程
- 归并排序：理解分治与归并，掌握 Merge 的正确实现
- 对比：同为 O(nlogn)，各算法的适用场景
- 练习：修复现有代码中的 Bug，实现性能统计

### 第三阶段：O(n) 非比较排序（1-2 周）

```
计数排序 → 基数排序 → 桶排序
```

**学习重点：**
- 理解非比较排序突破 O(nlogn) 下界的条件
- 计数排序：理解前缀和保证稳定性的原理
- 基数排序：理解"按位排序"思想，理解稳定性对基数排序的必要性
- 桶排序：理解均匀分布假设
- 练习：实现三种非比较排序，分析适用场景

### 第四阶段：工业级排序与扩展（1-2 周）

```
Timsort（理解原理） → Introsort（理解原理） → 外部排序（理解流程）
```

**学习重点：**
- Timsort：理解如何利用数据的局部有序性
- Introsort：理解 C++ `std::sort` 的实现策略
- 外部排序：理解磁盘 I/O 对算法设计的影响
- 练习：阅读 CPython 源码中的 Timsort 实现，撰写分析笔记

### 7.1 C++ STL 排序相关

| 组件 | 说明 | 参考文档 |

| `std::sort` | 基于 Introsort（内省排序），结合快速排序、堆排序和插入排序。C++ 标准不保证稳定性。平均 O(nlogn)，最坏 O(nlogn)（因退化时切换为堆排序） | [cppreference.com - std::sort](https://en.cppreference.com/w/cpp/algorithm/sort) |
| `std::stable_sort` | 稳定排序，通常基于归并排序实现。保证相等元素的原有顺序。时间复杂度 O(nlog²n)（无足够额外内存时）或 O(nlogn)（有足够额外内存时） | [cppreference.com - std::stable_sort](https://en.cppreference.com/w/cpp/algorithm/stable_sort) |
| `std::partial_sort` | 部分排序，仅保证前 k 个元素有序。基于堆排序实现 | [cppreference.com - std::partial_sort](https://en.cppreference.com/w/cpp/algorithm/partial_sort) |
| `std::nth_element` | 第 n 大元素选择，基于快速选择算法。平均 O(n) | [cppreference.com - std::nth_element](https://en.cppreference.com/w/cpp/algorithm/nth_element) |

> **时效性说明：**
> - C++11 起 `std::sort` 要求复杂度为 O(nlogn)（任何情况），此前标准仅要求平均 O(nlogn)
> - C++17 引入了并行版本 `std::execution::par`，可配合 `std::sort` 使用
> - C++20 引入了 `std::ranges::sort`，支持投影和范围操作
