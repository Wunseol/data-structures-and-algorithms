# 05 - 图模块需求文档

## 1. 模块概述

图（Graph）是计算机科学中最通用、最强大的数据结构之一。与线性表（一对一）和树（一对多）不同，图表达的是多对多的复杂关系。图论算法广泛应用于现实世界的方方面面：

- **地图与导航**：最短路径算法（Dijkstra、A*）驱动 GPS 导航系统
- **社交网络**：好友关系、社区发现、影响力传播分析
- **网络路由**：OSPF 协议使用 Dijkstra，BGP 使用路径向量算法
- **任务调度**：拓扑排序用于编译器依赖分析、构建系统（Makefile）
- **项目管理**：关键路径法（CPM）用于工程进度规划
- **芯片设计**：布线、布局问题建模为图问题
- **推荐系统**：图神经网络（GNN）基于图结构进行推荐

本模块为全新模块，项目当前无任何图相关代码。需从零构建完整的图数据结构与算法体系，以 C/C++ 为核心实现语言，Python 为辅助验证与可视化语言。

> **Python 实现**：参见 `python/algorithms/data_structures/` 中的 adjacency_matrix.py、adjacency_list.py、union_find.py

## 3. 新增需求

### 3.1 图的表示

#### 3.1.1 邻接矩阵（Adjacency Matrix）

**需求描述**：使用二维数组存储图的边关系，适合稠密图。

**实现要求**：

- 数据结构：
  ```c
  #define MAXV 100
  typedef struct {
      int edges[MAXV][MAXV];  // 邻接矩阵
      int n, e;               // 顶点数、边数
      int vexs[MAXV];         // 顶点数据
  } MGraph;
  ```
- 支持有向图/无向图
- 支持带权图/无权图
- 基本操作：创建、添加边、删除边、判断边是否存在、获取邻接点、获取度

**验收标准**：
- 正确创建有向/无向、带权/无权图
- 添加/删除边后矩阵状态正确
- 空间复杂度 O(V²)，适用于 V 较小的稠密图

#### 3.1.2 邻接表（Adjacency List）

**需求描述**：使用链表数组存储图的边关系，适合稀疏图。

**实现要求**：

- 数据结构（C 语言版本）：
  ```c
  typedef struct ArcNode {
      int adjvex;
      int weight;
      struct ArcNode *nextarc;
  } ArcNode;

  typedef struct VNode {
      int data;
      ArcNode *firstarc;
  } VNode;

  typedef struct {
      VNode adjlist[MAXV];
      int n, e;
  } ALGraph;
  ```
- 数据结构（C++ 版本，使用 STL）：
  ```cpp
  struct Edge {
      int to;
      int weight;
  };
  std::vector<std::vector<Edge>> adj;  // adj[u] = {Edge(v, w), ...}
  ```
- 支持有向图/无向图
- 支持带权图/无权图
- 基本操作：创建、添加边、删除边、判断边是否存在、获取邻接点、获取度

**验收标准**：
- 正确创建有向/无向、带权/无权图
- C 语言版本手动管理链表内存，无泄漏
- C++ 版本使用 `std::vector`，内存安全
- 空间复杂度 O(V+E)，适用于稀疏图
- 邻接矩阵与邻接表对同一图的操作结果一致

**技术参考**：
- [cppreference: std::vector](https://en.cppreference.com/w/cpp/container/vector) — C++11 起支持 `emplace_back`，C++20 起支持 `constexpr`
- C++ STL `std::vector` 作为邻接表容器是竞赛编程和工程实践中的标准做法

### 3.2 图的遍历

#### 3.2.1 深度优先搜索（DFS）

**需求描述**：实现 DFS 遍历，理解递归回溯思想。

**实现要求**：

- 递归实现
- 非递归实现（使用栈）
- 记录访问顺序
- 记录发现时间与完成时间（用于拓扑排序、强连通分量）
- 支持邻接矩阵和邻接表两种存储
- 应用：连通分量判定、环检测、路径查找

**验收标准**：
- 递归与非递归结果一致
- 正确检测有向图/无向图中的环
- 正确计算连通分量数
- 时间复杂度 O(V+E)（邻接表）/ O(V²)（邻接矩阵）

#### 3.2.2 广度优先搜索（BFS）

**需求描述**：实现 BFS 遍历，理解层次扩展思想。

**实现要求**：

- 使用队列实现
- 记录访问顺序与层次
- 支持邻接矩阵和邻接表两种存储
- 应用：无权图最短路径、连通分量、二分图判定

**验收标准**：
- BFS 访问顺序正确（按层次遍历）
- 无权图最短路径正确
- 正确判定二分图
- 时间复杂度 O(V+E)（邻接表）/ O(V²)（邻接矩阵）

**技术参考**：
- [cppreference: std::queue](https://en.cppreference.com/w/cpp/container/queue) — BFS 标准容器
- [cppreference: std::stack](https://en.cppreference.com/w/cpp/container/stack) — 非递归 DFS 容器

### 3.3 最小生成树

#### 3.3.1 Prim 算法

**需求描述**：从顶点出发，逐步扩展生成树，适合稠密图。

**实现要求**：

- 朴素实现：O(V²)
- 优先队列优化：O(E log V)，使用 `std::priority_queue`
- 输出最小生成树的边集与总权值
- 支持邻接矩阵和邻接表

**验收标准**：
- 生成树包含所有顶点且总权值最小
- 生成树边数为 V-1
- 与 Kruskal 算法结果一致（权值相同，边集可能不同）
- 处理不连通图时输出各连通分量的最小生成树（最小生成森林）

#### 3.3.2 Kruskal 算法

**需求描述**：从边出发，按权值排序逐步加入不构成环的边，适合稀疏图。

**实现要求**：

- 边排序：使用 `std::sort`
- 环检测：使用并查集（Union-Find）
- 输出最小生成树的边集与总权值

**验收标准**：
- 生成树包含所有顶点且总权值最小
- 生成树边数为 V-1
- 与 Prim 算法结果一致（权值相同）
- 时间复杂度 O(E log E)（排序主导）
- 并查集使用路径压缩 + 按秩合并

**技术参考**：
- [CLRS](https://mitpress.mit.edu/9780262046305/introduction-to-algorithms/) 第 23 章
- [cppreference: std::priority_queue](https://en.cppreference.com/w/cpp/container/priority_queue)

### 3.4 最短路径

#### 3.4.1 Dijkstra 算法

**需求描述**：求解单源最短路径，要求边权非负。

**实现要求**：

- 朴素实现：O(V²)
- 优先队列优化：O(E log V)（使用 `std::priority_queue` + 延迟删除或 decrease-key）
- 输出：从源点到各顶点的最短距离与路径
- 处理不可达顶点（输出 INF）

**验收标准**：
- 单源最短路径正确
- 不适用于负权边（需在文档中说明，并在代码中检测负权边时给出警告）
- 优先队列版本与朴素版本结果一致
- 提供 LeetCode 743（网络延迟时间）的解题验证

#### 3.4.2 Floyd 算法

**需求描述**：求解所有顶点对之间的最短路径。

**实现要求**：

- 三重循环实现：O(V³)
- 输出：任意两点间的最短距离矩阵
- 路径重建：记录前驱矩阵，可输出具体路径
- 检测负权环：对角线出现负值

**验收标准**：
- 所有顶点对最短路径正确
- 正确检测负权环
- 路径重建正确
- 与 Dijkstra 逐源求解结果一致（无非负权边时）

#### 3.4.3 Bellman-Ford 算法

**需求描述**：求解单源最短路径，支持负权边，可检测负权环。

**实现要求**：

- 基础实现：O(VE)
- SPFA 优化（队列优化版本）：平均 O(E)，最坏 O(VE)
- 输出：最短距离与路径，或报告存在负权环
- 应用：差分约束系统

**验收标准**：
- 正确处理负权边
- 正确检测负权环（第 V 次松弛仍有更新）
- SPFA 与基础版本结果一致
- 提供 LeetCode 787（K 站中转内最便宜的航班）的解题验证

**技术参考**：
- [CLRS](https://mitpress.mit.edu/9780262046305/introduction-to-algorithms/) 第 24-25 章
- [Algorithm Design Manual](https://www.algorist.com/)（Skiena）第 6 章 — 实践导向的图算法讲解

### 3.5 拓扑排序

**需求描述**：对有向无环图（DAG）进行拓扑排序。

**实现要求**：

- 基于入度的 Kahn 算法（BFS 风格）：O(V+E)
- 基于 DFS 后序逆序：O(V+E)
- 环检测：若排序结果不足 V 个顶点，则存在环
- 应用：课程安排、编译依赖、任务调度

**验收标准**：
- 拓扑排序结果满足所有前驱约束
- 正确检测有向图中的环
- Kahn 算法与 DFS 算法结果均为合法拓扑序列
- 提供 LeetCode 207（课程表）、210（课程表 II）的解题验证

### 3.6 关键路径

**需求描述**：在 AOE（Activity On Edge）网中求解关键路径。

**实现要求**：

- 计算事件最早发生时间 `ve[]`
- 计算事件最迟发生时间 `vl[]`
- 计算活动最早开始时间 `e[]` 和最迟开始时间 `l[]`
- 关键活动：`l[i] - e[i] == 0` 的活动
- 关键路径：由关键活动组成的路径
- 工期计算：`ve[汇点]`

**验收标准**：
- 关键路径正确
- 关键路径长度等于最短工期
- 非关键活动的松弛时间（`l - e`）正确
- 处理多条关键路径的情况

### 3.7 连通性问题

**需求描述**：求解无向图和有向图的连通性问题。

**实现要求**：

- 无向图：
  - 连通分量计数（DFS/BFS/并查集）
  - 割点（Articulation Point）：Tarjan 算法
  - 桥（Bridge）：Tarjan 算法
  - 双连通分量（BCC）

- 有向图：
  - 强连通分量（SCC）：Kosaraju 算法、Tarjan 算法
  - 缩点（将 SCC 缩为超级节点，得到 DAG）

**验收标准**：
- 连通分量计数正确
- 割点和桥识别正确
- 强连通分量划分正确
- Tarjan 算法时间复杂度 O(V+E)
- 提供 LeetCode 1192（查找集群内的关键连接）的解题验证

## 4. 实现语言

### 4.1 C/C++ 核心实现

- **C 语言版本**：手动管理内存，使用数组和指针实现邻接矩阵/邻接表，适合理解底层原理
- **C++ 版本**：使用 STL 容器（`std::vector`、`std::queue`、`std::priority_queue`、`std::stack`），代码更简洁安全
- **命名规范**：C 版本使用 `snake_case`，C++ 版本使用 `camelCase`
- **编译标准**：至少兼容 C++17，推荐 C++17

### 4.2 Python 辅助验证

- **可视化**：使用 `networkx` + `matplotlib` 绘制图形化图结构
- **正确性验证**：使用 `networkx` 内置算法对比验证 C/C++ 实现的正确性
- **测试数据生成**：使用 Python 生成随机图测试数据

**networkx 版本与兼容性**：
- 当前最新稳定版为 networkx 3.x 系列（要求 Python ≥ 3.10）
- networkx 3.0（2023 年 1 月）起移除了 Python 2 兼容代码，API 有较大变化
- networkx 3.2+（2023 年 10 月）支持 Python 3.12
- networkx 3.4+（2024 年）支持 Python 3.13
- 安装命令：`pip install networkx matplotlib`
- 参考 [networkx 官方文档](https://networkx.org/documentation/stable/)

**Python 验证示例**：
```python
import networkx as nx

G = nx.Graph()
G.add_edges_from([(0, 1), (1, 2), (2, 3)])
mst = nx.minimum_spanning_tree(G)
sp = nx.shortest_path(G, source=0, target=3)
```

## 7. 验收标准

### 7.1 代码质量验收

| 验收项 | 标准 |
|--------|------|
| 内存管理 | C 版本无内存泄漏（Valgrind/ASan 检测通过），C++ 版本使用 RAII |
| 接口统一 | 邻接矩阵和邻接表共享同一套算法接口（函数指针或模板） |
| 命名规范 | C 版本 `snake_case`，C++ 版本 `camelCase`，同一文件内不混用 |
| 错误处理 | 检测无效输入（负权边用于 Dijkstra、环用于拓扑排序等） |
| 编译警告 | GCC `-Wall -Wextra -Wpedantic` 零警告 |

### 7.2 功能验收

| 功能 | 验收标准 |
|------|---------|
| 邻接矩阵 | 正确创建/操作有向/无向/带权/无权图 |
| 邻接表 | 正确创建/操作有向/无向/带权/无权图，与邻接矩阵操作结果一致 |
| DFS | 递归与非递归结果一致，正确检测环和连通分量 |
| BFS | 按层次遍历，无权图最短路径正确 |
| Prim | 生成树权值最小，边数 V-1，与 Kruskal 权值一致 |
| Kruskal | 生成树权值最小，边数 V-1，与 Prim 权值一致 |
| Dijkstra | 单源最短路径正确，检测负权边警告 |
| Bellman-Ford | 正确处理负权边，检测负权环 |
| Floyd | 所有点对最短路径正确，检测负权环 |
| 拓扑排序 | 排序结果合法，检测环 |
| 关键路径 | 关键路径正确，工期计算正确 |
| 连通性 | 连通分量/强连通分量/割点/桥识别正确 |

### 7.3 性能验收

| 算法 | 邻接表时间复杂度 | 邻接矩阵时间复杂度 |
|------|-----------------|-------------------|
| DFS | O(V+E) | O(V²) |
| BFS | O(V+E) | O(V²) |
| Prim（优先队列） | O(E log V) | O(V²) |
| Kruskal | O(E log E) | O(E log E) |
| Dijkstra（优先队列） | O(E log V) | O(V²) |
| Bellman-Ford | O(VE) | O(V³) |
| Floyd | O(V³) | O(V³) |
| 拓扑排序 | O(V+E) | O(V²) |
