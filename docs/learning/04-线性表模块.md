# 线性表模块需求文档

## 1. 模块概述

线性表是最基本的数据结构，分为顺序表（基于数组）和链表（基于指针）两大类。本模块要求实现以下线性表类型：

| 子模块 | 目标路径 | 状态 |
|--------|---------|------|
| 顺序表 | [src/linear_list/sequential_list/](../../src/linear_list/sequential_list/) | 有多个版本，需整合修复 |
| 单链表 | [src/linear_list/singly_linked_list/](../../src/linear_list/singly_linked_list/) | 有多个版本，需整合修复 |
| 双向链表 | [src/linear_list/doubly_linked_list/](../../src/linear_list/doubly_linked_list/) | 已实现 |
| 循环链表 | [src/linear_list/circular_linked_list/](../../src/linear_list/circular_linked_list/) | 已实现 |

> **Python 实现**：参见 [python/algorithms/data_structures/](../../python/algorithms/data_structures/) 中的 [L01_sequential_list.py](../../python/algorithms/data_structures/L01_sequential_list.py)、[L04_singly_linked_list.py](../../python/algorithms/data_structures/L04_singly_linked_list.py)、[L05_doubly_linked_list.py](../../python/algorithms/data_structures/L05_doubly_linked_list.py)、[L06_circular_linked_list.py](../../python/algorithms/data_structures/L06_circular_linked_list.py)、[L09_skip_list.py](../../python/algorithms/data_structures/L09_skip_list.py)

## 2. 现有代码分析

### 2.1 顺序表

#### 2.1.1 [01_sequential_list_v1.cpp](../../src/linear_list/sequential_list/01_sequential_list_v1.cpp)（动态数组版）

**文件路径**：[../../src/linear_list/sequential_list/01_sequential_list_v1.cpp](../../src/linear_list/sequential_list/01_sequential_list_v1.cpp)

**功能概述**：菜单驱动的动态数组顺序表，支持初始化、插入、删除、查找、合并、交集、并集、逆置。

**Bug 与问题**：

**(1) 全局变量 L1-L4（第12-15行）**

```c
SqList L1;
SqList L2;
SqList L3;
SqList L4;
```

问题：使用全局变量，所有操作函数隐式依赖全局状态，无法操作多个独立顺序表实例。

**(2) SetupList 函数无返回值（第97-116行）**

```c
int SetupList()
{
    // ...
    // 缺少 return 语句
}
```

问题：函数声明返回 `int`，但函数体无 `return` 语句，属于未定义行为（UB）。参考：[cppreference - Undefined behavior](https://en.cppreference.com/w/c/language/behavior)

**(3) CheckLocate 函数逻辑错误（第118-135行）**

```c
int CheckLocate()
{
    int n,e;
    while(1){
        // ...
        if(n==1){
            printf("输出位序为:%d\n比较次数为:%d\n",Locate(L1,e),Locate(L1,e)-1);
            break;
        }else if(n==2){
            // ...
            break;
        }else{
            printf("输入错误,请重新输入!");
        }
        break;  // ← 第134行：此 break 导致 else 分支后也退出循环，使"重新输入"提示无效
    }
}
```

问题：第134行的 `break` 导致输入错误时也会退出循环，"请重新输入"提示形同虚设。

**(4) Merge 函数值传递导致数据丢失（第199-222行）**

```c
int Merge(SqList L1,SqList L2,SqList L3)  // ← 值传递
{
    // ...
    InsList(L3,k++,ai);  // 修改的是 L3 的副本，原 L3 不受影响
    // ...
}
```

问题：参数使用值传递，`InsList` 修改的是 `L3` 的副本，合并结果不会反映到全局 `L3`。应改为引用传递 `SqList &L3`。

**(5) Merge 函数中 k 的递增不一致（第205-208行）**

```c
if(ai<=aj){InsList(L3,k++,ai);++i;}     // k++：先插入位置k，再自增
else{InsList(L3,++k,aj);++j;}            // ++k：先自增，再插入位置k
```

问题：`k++` 和 `++k` 导致插入位置不一致。当 `ai<=aj` 时插入位置从0开始，当 `ai>aj` 时插入位置从1开始，导致合并结果错误。

**(6) Intersection 函数判断条件错误（第237行）**

```c
if(k==0){
    printf("无交集\n");
}
```

问题：`k` 初始化为1（第227行），即使无交集 `k` 也不等于0，判断条件永远为假。

**(7) InitList 函数无返回值（第305-312行）**

```c
int InitList(SqList &L,int s)
{
    // ...
    // 缺少 return 语句
}
```

问题：同 SetupList，声明返回 `int` 但无 `return`。

**(8) Locate 函数返回 false（第319-325行）**

```c
int Locate(const SqList &L,const int &e)
{
    for(int i=0;i<L.length;i++){
        if(L.data[i]==e)return i+1;
    }
    return false;  // ← false 为 0，与"位序从1开始"冲突
}
```

问题：`return false` 等价于 `return 0`，而位序从1开始，0表示"未找到"是合理的。但使用 `false` 语义不明确，应改为 `return 0` 并添加注释说明。

**(9) new 失败检查无效（第309行）**

```c
L.data=new int [s];
if(L.data==NULL)return false;  // ← new 失败时抛出 std::bad_alloc，不会返回 NULL
```

问题：C++ 中 `new` 失败时抛出异常而非返回 `NULL`。应使用 `new(std::nothrow)` 或 try-catch。参考：[cppreference - operator new](https://en.cppreference.com/w/cpp/memory/new/operator_new)

#### 2.1.2 [01_sequential_list_v2.c](../../src/linear_list/sequential_list/01_sequential_list_v2.c)（固定数组版）

**文件路径**：[../../src/linear_list/sequential_list/01_sequential_list_v2.c](../../src/linear_list/sequential_list/01_sequential_list_v2.c)

**功能概述**：使用全局固定数组 `arr[50]` 的顺序表实现，菜单驱动。

**Bug 与问题**：

**(1) malloc 内存泄漏（第29行）**

```c
if(num>50)
{
    (void*)malloc(2*num*sizeof(int));  // ← 返回值未赋给任何指针
}
```

问题：`malloc` 返回值被丢弃，分配的内存无法释放，造成内存泄漏。且即使分配了内存，后续操作仍使用全局 `arr[50]`，此段代码毫无意义。参考：[cppreference - malloc](https://en.cppreference.com/w/c/memory/malloc)

**(2) Check 函数每次调用 SetUp（第54行）**

```c
void Check()
{
    // ...
    int *p=SetUp();  // ← 每次查找都重新建表
    // ...
}
```

问题：查找操作不应重建顺序表。每次查找都要求用户重新输入数据，严重影响可用性。

**(3) Check 函数局部变量遮蔽全局变量（第55行）**

```c
void Check()
{
    // ...
    int *p=SetUp();
    int arr[50];  // ← 局部 arr 遮蔽全局 arr，且未使用
    // ...
}
```

问题：局部 `arr[50]` 遮蔽了全局 `arr[50]`，且该局部数组未被使用，浪费栈空间。

**(4) Delete 函数死循环（第139行）**

```c
if(num==count-1)
{
    printf("删除之后的顺序表：");
    for(i=0;i<=count-2;i--)  // ← i-- 应为 i++
    {
        printf("%d ",arr[i]);
    }
}
```

问题：循环条件 `i<=count-2` 配合 `i--`（从0开始递减），将导致死循环和数组越界访问。

**(5) Insert/Delete/Merge/Intersection/Sum/Reverse 中的 malloc 泄漏**

所有这些函数都存在与 SetUp 相同的 `malloc` 返回值丢弃问题（第85、128、176、195、226、244、299、316、327、389行）。

**(6) Sum 函数（并集）实现错误**

```c
for(i=0;arr3[i]!=-1;i++)
{
    if(arr3[i]==arr3[i+1])
    {
        arr3[i]=-1;  // ← 将重复元素标记为-1，但如果-1本身是有效数据则冲突
    }
}
```

问题：使用 `-1` 作为哨兵值，但未保证数据域不包含 `-1`；去重逻辑不正确，仅比较相邻元素，前提是数据已排序，但并集操作后数据不一定有序。

#### 2.1.3 [02_sequential_list_sorting.cpp](../../src/sorting/02_sequential_list_sorting.cpp)（排序算法集）

**文件路径**：[../../src/sorting/02_sequential_list_sorting.cpp](../../src/sorting/02_sequential_list_sorting.cpp)

**功能概述**：实现8种排序算法（直接插入、折半插入、希尔、冒泡、快速、简单选择、堆、归并），结构较清晰。

**Bug 与问题**：

**(1) Merge 函数边界条件错误（第195行）**

```c
void Merge(RedType R[],RedType T[],int low,int mid,int high)
{
    int i=low,j=mid+1,k=low;
    while(i<=mid&&j<high)  // ← 应为 j<=high
    // ...
    while(i<mid) T[k++]=R[i++];  // ← 应为 i<=mid
    while(j<high) T[k++]=R[j++]; // ← 应为 j<=high
}
```

问题：归并时右半部分的上界应为 `j<=high`，当前 `j<high` 会导致最后一个元素未被归并。同样，`i<mid` 应为 `i<=mid`。参考：归并排序标准实现见 [CLRS *Introduction to Algorithms*] 第2.3节。

**(2) 菜单文字错误（第312行）**

```c
case 8:
    cout<<"快速排序的结果为："<<endl;  // ← 应为"二路归并排序"
    MergeSort(L);
```

问题：归并排序的菜单提示文字写成了"快速排序"。

**(3) MSort 函数原地排序问题（第205-215行）**

```c
void MSort(RedType R[],RedType T[],int low,int high)
{
    // ...
    MSort(R,T,low,mid);
    MSort(R,T,mid+1,high);
    Merge(R,T,low,mid,high);  // ← R 和 T 是同一数组时，归并过程中覆写未处理数据
}
```

问题：`MergeSort` 调用 `MSort(L.r,L.r,1,L.length)`，源数组和目标数组相同。归并过程中 `T[k++]=R[i++]` 可能覆写尚未处理的 `R` 中数据，导致排序结果错误。标准做法应交替使用辅助数组。参考：[CLRS] 第2.3.1节。

#### 2.2.1 [04_singly_linked_list_v1.c](../../src/linear_list/singly_linked_list/04_singly_linked_list_v1.c)（基础版）

**文件路径**：[../../src/linear_list/singly_linked_list/04_singly_linked_list_v1.c](../../src/linear_list/singly_linked_list/04_singly_linked_list_v1.c)

**功能概述**：带头结点的单链表，支持头插法/尾插法建表、求表长、按值查找、插入、删除、输出。

**Bug 与问题**：

**(1) ClearList 函数名与功能不符（第138-153行）**

```c
void ClearList(LinkList *head)  // ← 函数名为"清空"，实际执行的是"逆置"但实现也是清空
{
    LinkList *p,*q;
    p=head->next;
    while(p){
        q=p->next;
        // delete p;  ← 释放代码被注释
        p=q;
    }
    head->next=NULL;
    if(p==NULL&&head->next==NULL){
        printf("链表head逆置成功！");  // ← 提示"逆置成功"，但实际是清空
    }
}
```

问题：
- 函数名为 `ClearList`（清空），但菜单中标注为"逆置单链表"（第179行），提示信息也写"逆置成功"
- `delete p` 被注释掉，导致节点内存未释放
- 实际效果是清空链表（将 `head->next` 置 NULL），而非逆置

**(2) DestoryList 函数名拼写错误（第156行）**

```c
void DestoryList(LinkList *head)  // ← 应为 DestroyList
```

问题：`Destory` 应为 `Destroy`，拼写错误。

**(3) DestoryList 函数内存泄漏（第156-169行）**

```c
void DestoryList(LinkList *head)
{
    LinkList *p;
    while(head){
        p=head;
        head=head->next;
        // delete p;  ← 释放代码被注释
    }
}
```

问题：`delete p` 被注释，链表节点内存未释放。且此文件为 `.c` 文件，应使用 `free(p)` 而非 `delete p`。

**(4) 菜单输入处理问题（第194行）**

```c
scanf("%c",&ch2);
```

问题：`scanf("%c")` 会读取前一次输入残留的换行符，导致菜单选择跳过。应使用 `scanf(" %c",&ch2)`（加空格跳过空白字符）。参考：[cppreference - scanf](https://en.cppreference.com/w/c/io/fscanf)

**文件路径**：[../../src/linear_list/singly_linked_list/04_singly_linked_list_v2.c](../../src/linear_list/singly_linked_list/04_singly_linked_list_v2.c)

**功能概述**：带头结点的单链表，使用随机数据生成，支持初始化、头插法/尾插法建表、求表长、按位置/按值查找、插入、删除、清空、逆置、输出。

**Bug 与问题**：

**(1) struct 声明中指针类型拼写错误（第18行）**

```c
typedef struct Node
{
    ElemType data;
    struct Note *next;  // ← 应为 struct Node *next
}Node;
```

问题：`struct Note` 应为 `struct Node`，拼写错误。在某些编译器中可能因前向引用而通过编译，但属于未定义行为。

**(2) InitList 函数返回类型错误（第45-53行）**

```c
Status InitList()  // ← 返回类型为 Status (int)
{
    LinkList L;
    L=(Node*)malloc(sizeof(Node));
```

问题：返回类型声明为 `Status`（`int`），但实际返回 `LinkList`（指针），类型不匹配。指针转为 `int` 可能导致数据截断（64位系统上指针为8字节，int为4字节）。

**(3) InitList 函数未更新全局 L1（第45-53行）**

```c
Status InitList()
{
    LinkList L;  // ← 局部变量
    L=(Node*)malloc(sizeof(Node));
    // ...
    return L;  // ← 返回局部 L，但全局 L1 未被更新
```

问题：`InitList` 操作的是局部变量 `L`，全局 `L1` 未被初始化。后续菜单中使用 `L1` 时将导致未定义行为。

**(4) CreateListTail 函数错误赋值（第168行）**

```c
r->data=NULL;  // ← 应为 r->next=NULL
```

问题：`r->data=NULL` 将尾节点的数据域设为0（NULL），而非将指针域设为空。应改为 `r->next=NULL`。

**(5) ListDelete 函数循环条件错误（第126行）**

```c
while(p->next&&j<1){  // ← 应为 j<i
    p=p->next;
    ++j;
}
```

问题：`j<1` 应为 `j<i`，导致循环最多执行0次，无法定位到正确的删除位置。

**(6) Reverse 函数实现错误（第186-201行）**

```c
void Reverse(LinkList L)
{
    LinkList p,q;
    p=L->next;
    while(p){
        q=p->next;
        p=q;  // ← 仅移动指针，未执行逆置操作
    }
    L->next=NULL;
    // ...
}
```

问题：逆置函数仅遍历链表并将 `L->next` 置空，未执行任何逆置操作（如修改指针方向）。正确实现应逐个断开节点并头插到新链表。

**(7) 菜单大量功能被注释（第246-260行）**

```c
// case 'd':
// 	Delete();
// 	break;
// case 'e':
// 	MergeList();

// ...
```

问题：菜单中 d-i 选项的实现被注释，导致大部分功能不可用。

**功能概述**：C++ 实现的带头结点单链表，支持初始化、显示、末尾插入、查找、删除、插入排序、逆置、合并、拆分。

**Bug 与问题**：

**(1) insert 函数重新指向头结点（第220行）**

```c
void insert(Node *DLList)
{
    Node *p,*s;
    DLList=&Head;  // ← 参数传入的指针被覆盖，失去通用性
    // ...
}
```

问题：函数参数 `DLList` 被重新赋值为 `&Head`，使函数无法操作其他链表。`search`、`del`、`reverse`、`GetLengthList`、`ListInsert` 函数也存在相同问题。

**(2) split 函数按数据值奇偶拆分而非位序（第415-443行）**

```c
void split(Node *DLList1,Node *DLList2)
{
    // ...
    if(p->next->data % 2 == 0){  // ← 按数据值奇偶拆分
        // ...
    }
}
```

问题：函数注释要求"按序号为奇偶拆分"，但实现按数据值的奇偶拆分，与需求不符。

**(3) MergeList 函数合并后未去重（第373-411行）**

```c
void MergeList(Node *DLList1,Node *DLList2)
{
    // ...
    if(p->data<q->data)
    {
        r->next=p;
        r=p;
        p=p->next;
    }
    else  // ← p->data >= q->data 时，未判断是否相等
    {
        r->next=q;
        r=q;q=q->next;
    }
}
```

问题：当两个链表有相同元素时，未做去重处理，合并后链表可能包含重复元素。

### 5.2 循环单链表

**目标路径**：[src/linear_list/circular_linked_list/](../../src/linear_list/circular_linked_list/)

**必须实现的操作**：

| 操作 | 函数签名 | 说明 |
|------|---------|------|
| 初始化 | `CiLinkList InitCiList()` | 创建带头结点的空循环单链表 |
| 尾插法建表 | `void CreateCiListTail(CiLinkList *L, int n)` | 尾插法，最后节点next指向头结点 |
| 插入 | `Status ListInsert(CiLinkList *L, int i, ElemType e)` | 在第i个位置前插入 |
| 删除 | `Status ListDelete(CiLinkList *L, int i, ElemType *e)` | 删除第i个元素 |
| 判空 | `Status ListEmpty(CiLinkList L)` | `L->next == L` 时为空 |
| 求表长 | `int ListLength(CiLinkList L)` | 循环终止条件为 `p != L` |
| 遍历 | `void ListTraverse(CiLinkList L)` | 循环遍历 |
| 合并 | `CiLinkList MergeCiList(CiLinkList La, CiLinkList Lb)` | 合并两个循环链表 |

**注意事项**：
- 循环链表中尾节点的 `next` 指向头结点（而非 NULL）
- 遍历终止条件为 `p->next != L`（而非 `p != NULL`）
- 常使用尾指针（而非头指针）来操作循环链表，可 O(1) 访问首尾节点
