# 08 查找算法模块需求文档

## 1. 模块概述

查找（搜索）算法是数据结构与算法中最基础、最常用的操作之一。几乎所有数据结构的核心操作都包含查找：顺序表的按值定位、链表的遍历查找、二叉排序树的搜索、哈希表的键值查询、图的路径搜索等。本模块旨在系统性地建立查找算法知识体系，从基础的线性查找和二分查找，到高级的插值查找、斐波那契查找，再到树形查找和哈希查找，并建立与其他模块的交叉引用关系。

**模块目标：**
- 掌握线性查找与二分查找的原理与实现
- 修复现有代码中的查找相关问题
- 补充二分查找的变体（lower_bound、upper_bound、旋转数组查找）
- 实现插值查找和斐波那契查找
- 建立与其他模块的交叉引用（BST 查找 → 树模块，哈希查找 → 哈希表模块，DFS/BFS → 图模块，KMP → 字符串模块）
- 理解 C++ STL 和 Python 标准库中的查找工具

**实现语言：** C/C++（主要实现语言），Python（辅助验证与对比）

> **Python 实现**：参见 [python/algorithms/searching/](../../python/algorithms/searching/)（4 种搜索算法）

### 2.2 线性查找分析

#### 01_sequential_list_v1.cpp 中的 Locate 函数

```cpp
int Locate(const SqList &L,const int &e)
{
    for(int i=0;i<L.length;i++){
        if(L.data[i]==e)return i+1;
    }
    return false;
}
```

**问题分析：**
- 返回值语义混乱：成功时返回位序（1-based），失败时返回 `false`（即 0），但位序 0 在 1-based 体系中是无效的，导致无法区分"找到第 0 个元素"和"未找到"
- 实际上由于是 1-based 位序，0 表示未找到是可行的，但 `return false` 的写法不直观，应改为 `return 0`
- 未统计比较次数
- 仅返回第一个匹配位置，不支持查找所有匹配

#### 01_sequential_list_v2.c 中的 Check 函数


### 2.3 二分查找分析

#### 03_search_main.cpp 中的 binarySearch 函数

```cpp
void binarySearch(SqList &list)
{
    printf("折半查找条件保证要好排序的\n");
    simpleSort(list);  // 副作用：每次查找都排序
    int low,high,mid,e,flag=0;
    count1=0;
    low=0;
    high=list.length;  // Bug: 应为 list.length-1
    printf("请从键盘输入一个元素值:");
    scanf("%d",&e);
    while(low<=high){
        mid=(low+high)/2;
        if(e<list.elem[mid]){
            count1++;
            high=mid-1;
        }else if(e>list.elem[mid]){
            count1++;
            low=mid+1;
        }else{
            if(e==list.elem[mid]){
                flag=1;
                printf("找到元素%d\n",e);
                printf("输出位序为:%d\n比较次数为:%d\n",mid+1,count1+1);
            }
            break;
        }
    }
    if(low>high&&flag==0){
        printf("输入错误,未找到元素%d\n",e);
    }
    system("pause");
}
```

**严重问题：**

1. **副作用问题（P0）：** `binarySearch` 函数内部调用 `simpleSort(list)` 对原表进行排序，这违反了查找操作不应修改数据的原则。用户可能不希望仅因为查找就改变数据顺序。
   - **修复方案：** 方案 A —— 要求调用前确保数据已排序，若未排序则提示用户先排序；方案 B —— 创建副本进行排序和查找，不修改原表

2. **边界错误（P1）：** `high=list.length` 应为 `high=list.length-1`，因为数组下标从 0 开始，`list.elem[list.length]` 越界访问

3. **比较次数统计不准确（P2）：** 仅在 `e<list.elem[mid]` 和 `e>list.elem[mid]` 时增加计数，但 `e==list.elem[mid]` 时也进行了一次比较，且 `else` 分支中又做了一次 `if(e==list.elem[mid])` 的冗余比较

4. **冗余判断（P2）：** `else` 分支已经保证 `e==list.elem[mid]`，内部的 `if(e==list.elem[mid])` 判断是冗余的

5. **提示信息不准确（P3）：** "输入错误"应为"未找到元素"

### 2.4 BST 查找分析

#### 11_bst_crud.cpp 中的 SearchBstNode 函数

```cpp
BstNode* SearchBstNode(BstNode* T, const TElemType key)
{
    BstNode *p = T;
    if (NULL == p)
        return NULL;
    if (p->data == key)
        p = T;
    else if (p->data > key)
        p = SearchBstNode(T->Left, key);
    else
        p = SearchBstNode(T->Right, key);
    return p;
}
```

**问题分析：**
- `p->data == key` 时 `p = T` 是冗余的（p 已经等于 T），应直接 `return p`
- 实现正确但可优化为迭代版本以避免栈溢出
- 此函数属于树模块，本模块仅做交叉引用

### 3.2 插值查找（Interpolation Search）

- **原理：** 基于二分查找的改进，根据目标值在有序序列中的大致位置进行插值估计
- **估计公式：** `mid = low + (key - arr[low]) * (high - low) / (arr[high] - arr[low])`
- **时间复杂度：** 平均 O(loglogn)，最坏 O(n)（数据分布不均匀时）
- **适用场景：** 数据分布均匀且有序的表
- **实现要求：**
  - 基本插值查找实现
  - 与二分查找的性能对比
  - 分析数据分布对性能的影响
- **注意事项：** 当 `arr[high] == arr[low]` 时需特殊处理，防止除零错误

### 3.3 斐波那契查找（Fibonacci Search）

- **原理：** 利用斐波那契数列对有序表进行分割，是二分查找的变体
- **分割策略：** 将表分为长度为 F(k-1)-1 和 F(k-2)-1 的两部分
- **时间复杂度：** 平均 O(logn)，最坏 O(logn)
- **与二分查找的区别：**
  - 分割点不是中点，而是由斐波那契数列决定
  - 只涉及加减法，不涉及除法（在某些硬件上有优势）
  - 后半部分总比前半部分多一个元素或相等
- **实现要求：**
  - 预计算斐波那契数列
  - 处理表长不为 F(k)-1 的情况（补齐）
  - 与二分查找的性能对比

## 5. 学习路径

### 第一阶段：线性查找（1 周）

```
基本线性查找 → 带哨兵线性查找 → 线性查找优化
```

**学习重点：**
- 理解顺序查找的基本思想和 ASL（平均查找长度）计算
- 哨兵的作用：减少循环内的比较次数
- 线性查找的适用场景：无序表、链表、小规模数据
- ASL 计算：成功 ASL = (n+1)/2，失败 ASL = n

### 第二阶段：二分查找及其变体（2 周）

```
标准二分查找 → lower_bound/upper_bound → 旋转数组查找
```

**学习重点：**
- 二分查找的前提条件：有序、随机访问
- 循环不变量的理解：`[low, high]` 闭区间 vs `[low, high)` 半开区间
- lower_bound 和 upper_bound 的精确定义与实现
- 旋转数组查找：如何判断有序半边
- 二分查找的常见 Bug：溢出（`mid = low + (high - low) / 2`）、边界条件
- ASL 计算：成功 ASL ≈ log₂(n+1) - 1，失败 ASL ≈ log₂n

### 第三阶段：高级查找（1-2 周）

```
插值查找 → 斐波那契查找 → 查找算法对比
```

**学习重点：**
- 插值查找：理解插值公式，分析数据分布对性能的影响
- 斐波那契查找：理解斐波那契分割策略，与二分查找的对比
- 三种 O(logn) 查找的对比：二分（通用）、插值（均匀分布优）、斐波那契（无除法优）

### 第四阶段：跨模块查找（持续）

```
BST 查找（树模块） → 哈希查找（哈希表模块） → DFS/BFS（图模块） → 字符串查找（字符串模块）
```

**学习重点：**
- BST 查找：O(logn) 平均，O(n) 最坏，AVL/红黑树保证 O(logn)
- 哈希查找：O(1) 平均，理解哈希冲突对性能的影响
- DFS/BFS：图/状态空间中的搜索策略
- 字符串查找：模式匹配的特殊查找问题

> - C++20 引入了 `std::ranges` 命名空间下的查找算法，如 `std::ranges::find`、`std::ranges::lower_bound`、`std::ranges::binary_search` 等，支持投影（projection）和范围操作
> - C++20 新增 `std::ranges::search` 支持 Searcher 策略，包括默认搜索器、Boyler-Moore 搜索器（`std::boyer_moore_searcher`）和 Boyer-Moore-Horspool 搜索器（`std::boyer_moore_horspool_searcher`）
> - C++23 新增 `std::ranges::contains` 和 `std::ranges::contains_subrange`，简化存在性检查
> - C++26 计划引入更多并行算法和 SIMD 友好的查找算法
