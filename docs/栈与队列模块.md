# 栈与队列模块需求文档

## 1. 模块概述

栈（Stack）和队列（Queue）是两种重要的受限线性表。栈遵循后进先出（LIFO）原则，队列遵循先进先出（FIFO）原则。本模块要求实现以下类型：

| 子模块 | 目标路径 | 状态 |
|--------|---------|------|
| 顺序栈 | `src/stack/sequential_stack/` | 有多个版本，含严重bug，需修复 |
| 链栈 | `src/stack/linked_stack/` | 有1个版本，含逻辑bug，需修复 |
| 顺序队列 | `src/queue/sequential_queue/` | **完全缺失** |
| 循环队列 | `src/queue/circular_queue/` | **完全缺失** |
| 链队列 | `src/queue/linked_queue/` | **完全缺失** |
| 双端队列 | `src/queue/deque/` | **完全缺失** |
| 优先队列 | `src/queue/priority_queue/` | **完全缺失** |

## 2. 现有栈代码分析

### 2.1 `Stack1.cpp` 和 `Stack2.cpp`（顺序栈）

**文件路径**：`Stack栈/Stack1.cpp`、`Stack栈/Stack2.cpp`

**说明**：两个文件内容完全相同，以下分析以 `Stack1.cpp` 为准。

**功能概述**：动态分配的顺序栈，支持初始化、判空、入栈、出栈、取栈顶、打印。

**Bug 与问题**：

#### Bug-1：GetTop 函数 if-else 缺少大括号（第58-64行）

```c
Status GetTop(Sqstack &st,int &e)
{
    if(st.top==st.base)
        return ERROR;
        e=*(st.top-1);   // ← 此行不属于 if 分支，无论栈是否为空都会执行
        return OK;        // ← 此行也不属于 if 分支，总是返回 OK
}
```

**问题分析**：由于 `if` 语句后没有大括号，只有 `return ERROR` 属于 `if` 分支。当栈为空时，`return ERROR` 会提前返回，所以 `e=*(st.top-1)` 不会在栈空时执行。但当栈不为空时，函数能正常工作。因此这个 bug 在运行时不会触发错误，但代码意图不清晰，违反了防御性编程原则。

**修复方案**：

```c
Status GetTop(Sqstack &st, int &e)
{
    if (st.top == st.base)
        return ERROR;
    e = *(st.top - 1);
    return OK;
}
```

参考：[cppreference - if statement](https://en.cppreference.com/w/c/language/if)，建议始终使用大括号。

#### Bug-2：StackEnter 函数使用赋值代替比较（第32行）

```c
Status StackEnter(Sqstack st,int &e)
{
    if(st.top==st.base)
        return ERROR;
    else if(e=*(st.top-1))  // ← 赋值运算符 = 而非比较运算符 ==
        return OK;
}
```

**问题分析**：
1. `e=*(st.top-1)` 是赋值操作，将栈顶元素赋给 `e`，而非比较
2. 赋值表达式的值为赋值后的 `e`，如果栈顶元素非0则返回 `OK`，为0则不返回任何值（缺少 `return` 语句）
3. 函数名为 `StackEnter`（入栈？），但注释和菜单中标注为"判断栈是否为空"，命名严重误导
4. 参数 `Sqstack st` 为值传递，对 `st` 的修改不影响原栈
5. 函数末尾缺少 `return` 语句

**修复方案**：

```c
Status StackEmpty(Sqstack st)
{
    if (st.top == st.base)
        return ERROR;
    return OK;
}
```

参考：[cppreference - assignment operator](https://en.cppreference.com/w/c/language/operator_assignment)，赋值与比较混淆是 C 语言最经典的 bug 之一。

#### Bug-3：Printfstack 函数修改 base 指针（第66-78行）

```c
Status Printfstack(Sqstack &st)
{
    if(st.base==st.top){
        printf("空栈！\n");
        return ERROR;
    }
    else
        printf("栈为：");
        while(st.base!=st.top){   // ← 此行不属于 else 分支
            printf("%d ",*st.base);
            st.base++;            // ← 修改了 base 指针！
        }
}
```

**问题分析**：
1. `else` 后缺少大括号，`while` 循环不属于 `else` 分支，即使栈为空也会执行
2. `st.base++` 修改了栈的 `base` 指针，打印后栈结构被破坏，后续操作将产生未定义行为
3. 函数末尾缺少 `return` 语句

**修复方案**：

```c
Status Printfstack(Sqstack &st)
{
    if (st.base == st.top) {
        printf("空栈！\n");
        return ERROR;
    }
    printf("栈为：");
    int *p = st.base;
    while (p != st.top) {
        printf("%d ", *p);
        p++;
    }
    printf("\n");
    return OK;
}
```

#### Bug-4：Push 函数入栈 -1 后继续入栈（第115-122行）

```c
case 'c':
    printf("请输入要进栈的元素(输入-1结束):");
    while(e!=-1){
        scanf("%d",&e);
        if(Push(stack1,e))         // ← -1 也会被入栈
            printf("%d入栈成功!\n",e);
        else
            printf("%d入栈失败!\n",e);
    }
```

**问题分析**：先判断 `e!=-1` 再读取输入，但读取后未判断是否为 -1 就直接入栈。-1 会被入栈后才退出循环。

**修复方案**：

```c
while (1) {
    scanf("%d", &e);
    if (e == -1) break;
    if (Push(stack1, e))
        printf("%d入栈成功!\n", e);
    else
        printf("%d入栈失败!\n", e);
}
```

#### Bug-5：case 'f' 缺少大括号（第141-143行）

```c
case 'f':
    if(Printfstack(stack1))
        printf("打印完成！\n");
        break;       // ← break 不属于 if，但此处恰好行为正确
```

**问题分析**：`break` 不在 `if` 的大括号内，但由于 `break` 无论 `if` 条件是否成立都应执行，此处运行时行为正确，但代码风格不规范。

#### Bug-6：StackEnter 值传递问题（第28行）

```c
Status StackEnter(Sqstack st,int &e)  // ← st 为值传递
```

**问题分析**：`Sqstack st` 为值传递，函数内对 `st` 的修改不影响原栈。虽然此函数仅读取栈状态，但值传递意味着整个结构体被拷贝（包括 `base`、`top` 指针和 `stacksize`），效率低下。应改为 `const Sqstack &st`。

#### Bug-7：case 1 中变量声明位置（第109行）

```c
case 1:

```

**问题分析**：在 C++ 中，`switch-case` 中直接声明变量可能导致跳过初始化的问题。应在 `case` 内添加大括号或移至 `switch` 之前。参考：[cppreference - switch statement](https://en.cppreference.com/w/cpp/language/switch)

### 2.3 `s2.cpp`（链栈）

**文件路径**：`Stack栈/s2.cpp`

**功能概述**：不带头结点的链栈实现，支持初始化、判空、入栈、出栈、取栈顶、打印。

**Bug 与问题**：

#### Bug-8：StackEnter 返回值逻辑反转（第30-36行 + 第108-113行）

```c
Status StackEnter(LinkStack st)
{
    if(st==NULL)
        return TRUE;   // ← 栈空返回 TRUE
    else
        return FALSE;  // ← 栈非空返回 FALSE
}
```

```c
case 1:
    int y;
    if(StackEnter(st))          // ← StackEnter 返回 TRUE 表示栈空
    {
        printf("栈不为空！\n"); // ← 但这里打印"栈不为空"
        printf("栈顶元素为%d\n",y);  // ← y 未初始化
    }
```

**问题分析**：
1. `StackEnter` 函数：栈空返回 `TRUE`，栈非空返回 `FALSE`
2. `main` 函数中：`if(StackEnter(st))` 为真时打印"栈不为空"——逻辑完全反转
3. 变量 `y` 未初始化就使用，输出未定义值
4. 函数名 `StackEnter`（入栈？）与功能（判空）严重不符

**修复方案**：

```c
Status StackEmpty(LinkStack st)
{
    if (st == NULL)
        return TRUE;
    return FALSE;
}

// main 中：
case 1:
    if (StackEmpty(st)) {
        printf("栈为空！\n");
    } else {
        printf("栈不为空！\n");
        printf("栈顶元素为%d\n", GetTop(st));
    }
    break;
```

#### Bug-9：GetTop 函数返回值语义混乱（第66-70行）

```c
Status GetTop(LinkStack &st)
{
    if(st!=NULL)
        return st->data;  // ← 返回栈顶数据值，而非状态码
}
```

**问题分析**：
1. 函数返回类型为 `Status`（`int`），但返回的是栈顶数据值
2. 如果栈顶数据值为0，无法区分"栈顶为0"和"栈空"
3. 栈空时无返回值（缺少 `return` 语句）
4. 在 main 中（第132-135行）：`if(GetTop(st))` 用返回值判断是否成功，但如果栈顶元素为0，条件为假，会错误地认为获取失败

**修复方案**：

```c
Status GetTop(LinkStack st, SElemType &e)
{
    if (st == NULL)
        return ERROR;
    e = st->data;
    return OK;
}
```

#### Bug-10：Printfstack 函数 else 缺少大括号（第74-86行）

```c
Status Printfstack(LinkStack &st) {
    if(st==NULL){
        printf("空栈！\n");
        return ERROR;
    }
    else
        printf("栈的内容为：");
        LinkStack p=st;       // ← 不属于 else
        while(p!=NULL){
            printf("%d ",p->data);
            p=p->next;}
            printf("\n");
}
```

**问题分析**：`else` 后缺少大括号，`LinkStack p=st` 及后续代码不属于 `else` 分支。当栈为空时，`return ERROR` 提前返回，所以运行时不会出错，但代码意图不清晰。

#### Bug-11：Pop 函数使用 delete 而非 free（第52-62行）

```c
Status Pop(LinkStack &st,int &e)
{
    // ...
    delete p;  // ← 文件扩展名为 .cpp，语法正确
    return OK;
}
```

**问题分析**：文件为 `.cpp`，使用 `delete` 语法正确。但 `Push` 函数（第42行）使用 `new` 分配内存，`Pop` 使用 `delete` 释放，配对正确。此处不是 bug，但需注意 C/C++ 混用风格的一致性。

#### Bug-12：main 函数返回值（第146行）

```c
return 1;  // ← 应为 return 0
```

**问题分析**：`main` 函数返回非0值通常表示异常退出。参考：[cppreference - main function](https://en.cppreference.com/w/cpp/language/main_function)

## 5. 队列新增需求

### 5.1 顺序队列

**目标路径**：`src/queue/sequential_queue/`

**必须实现的操作**：

| 操作 | 函数签名 | 说明 |
|------|---------|------|
| 初始化 | `Status InitQueue(SqQueue *Q)` | 分配 MAXSIZE 大小的数组 |
| 销毁 | `Status DestroyQueue(SqQueue *Q)` | 释放数组内存 |
| 清空 | `Status ClearQueue(SqQueue *Q)` | 重置 front 和 rear |
| 判空 | `Status QueueEmpty(SqQueue Q)` | `front == rear` 时为空 |
| 求队长 | `int QueueLength(SqQueue Q)` | `rear - front` |
| 入队 | `Status EnQueue(SqQueue *Q, QElemType e)` | 尾部插入 |
| 出队 | `Status DeQueue(SqQueue *Q, QElemType *e)` | 头部删除 |
| 取队头 | `Status GetHead(SqQueue Q, QElemType *e)` | 获取队头元素 |
| 遍历 | `Status QueueTraverse(SqQueue Q)` | 从队头到队尾遍历 |

**数据结构定义**：

```c
#define MAXQSIZE 100
typedef struct {
    QElemType *base;
    int front;
    int rear;
} SqQueue;
```

**注意事项**：
- 顺序队列存在"假溢出"问题：`rear` 达到 `MAXQSIZE` 但 `front` 前有空位
- 建议直接实现循环队列替代纯顺序队列

### 5.2 循环队列

**目标路径**：`src/queue/circular_queue/`

**必须实现的操作**：

| 操作 | 函数签名 | 说明 |
|------|---------|------|
| 初始化 | `Status InitQueue(SqQueue *Q)` | 分配 MAXQSIZE 大小数组 |
| 判空 | `Status QueueEmpty(SqQueue Q)` | `front == rear` |
| 判满 | `Status QueueFull(SqQueue Q)` | `(rear+1) % MAXQSIZE == front` |
| 求队长 | `int QueueLength(SqQueue Q)` | `(rear - front + MAXQSIZE) % MAXQSIZE` |
| 入队 | `Status EnQueue(SqQueue *Q, QElemType e)` | `Q->rear = (Q->rear+1) % MAXQSIZE` |
| 出队 | `Status DeQueue(SqQueue *Q, QElemType *e)` | `Q->front = (Q->front+1) % MAXQSIZE` |
| 取队头 | `Status GetHead(SqQueue Q, QElemType *e)` | 返回 `base[front]` |
| 遍历 | `Status QueueTraverse(SqQueue Q)` | 循环遍历 |

**关键设计决策**：
- 牺牲一个存储单元区分空/满：`front == rear` 为空，`(rear+1)%MAXQSIZE == front` 为满
- 或使用计数器 `size` 字段区分空/满（推荐，更直观）

参考：[CLRS *Introduction to Algorithms*] 10.1节；严蔚敏《数据结构》3.5节。

### 5.3 链队列

**目标路径**：`src/queue/linked_queue/`

**必须实现的操作**：

| 操作 | 函数签名 | 说明 |
|------|---------|------|
| 初始化 | `Status InitQueue(LinkQueue *Q)` | 创建头结点，front/rear 指向头结点 |
| 销毁 | `Status DestroyQueue(LinkQueue *Q)` | 释放所有节点 |
| 判空 | `Status QueueEmpty(LinkQueue Q)` | `Q.front == Q.rear` |
| 入队 | `Status EnQueue(LinkQueue *Q, QElemType e)` | 尾插法 |
| 出队 | `Status DeQueue(LinkQueue *Q, QElemType *e)` | 头删法 |
| 取队头 | `Status GetHead(LinkQueue Q, QElemType *e)` | 返回 `front->next->data` |
| 求队长 | `int QueueLength(LinkQueue Q)` | 遍历计数 |
| 遍历 | `Status QueueTraverse(LinkQueue Q)` | 从队头到队尾 |

**数据结构定义**：

```c
typedef struct QNode {
    QElemType data;
    struct QNode *next;
} QNode, *QueuePtr;

typedef struct {
    QueuePtr front;
    QueuePtr rear;
} LinkQueue;
```

### 5.4 双端队列

**目标路径**：`src/queue/deque/`

**必须实现的操作**：

| 操作 | 函数签名 | 说明 |
|------|---------|------|
| 初始化 | `Status InitDeque(Deque *D)` | 创建空双端队列 |
| 头部插入 | `Status PushFront(Deque *D, DequeElemType e)` | 队头插入 |
| 尾部插入 | `Status PushBack(Deque *D, DequeElemType e)` | 队尾插入 |

### 5.5 优先队列

## 6. 栈的应用场景需求

### 6.1 表达式求值

**目标路径**：`src/stack/sequential_stack/expression_eval.cpp`

**功能要求**：
- 支持中缀表达式求值（含 `+`、`-`、`*`、`/`、`(`、`)`）
- 使用双栈法（操作数栈 + 运算符栈）
- 支持多位整数和负数
- 输入验证：括号匹配、非法字符检测

**算法参考**：Dijkstra 双栈算法，见 [Algorithms 4th Edition, Sedgewick & Wayne] Section 1.3。

### 6.2 括号匹配

**目标路径**：`src/stack/sequential_stack/parenthesis_match.cpp`

**功能要求**：
- 支持 `()`、`[]`、`{}` 三种括号
- 输入一个字符串，输出是否匹配
- 报告不匹配的位置和原因（缺少左括号/右括号/类型不匹配）

**算法**：逐字符扫描，遇左括号入栈，遇右括号与栈顶比较。

### 6.3 函数调用栈模拟

- 展示函数调用时栈帧的入栈和出栈

### 6.4 撤销/重做

**目标路径**：`src/stack/sequential_stack/undo_redo.cpp`

**功能要求**：

- 撤销操作将状态从"操作栈"弹出到"重做栈"

### 8.4 时效性注意事项

| 注意项 | 说明 |
|--------|------|
| C11 `<threads.h>` | MSVC 截至 2025 年仍未完整支持 C11 `<threads.h>`。若需跨平台多线程，建议使用 C++11 `<thread>` 或 POSIX `pthread` |
| C++20 `std::jthread` | C++20 引入 `std::jthread`，析构时自动 join，避免资源泄漏。建议 C++20 环境下优先使用 |
| C++23 `std::stacktrace` | C++23 引入 `<stacktrace>` 头文件，可直接获取调用栈，可用于函数调用栈模拟的可视化 |
| C23 `nullptr` | C23 引入 `nullptr` 关键字替代 `NULL`，类型安全。当前项目 C 代码仍使用 `NULL` |
| `realloc` 行为 | C17 标准下 `realloc(ptr, 0)` 的行为是实现定义的（可能返回空指针或释放内存），C23 明确了其行为。建议避免 `realloc(ptr, 0)` |
